# 디자인 웹(AI 디자인 생성)용 프롬프트 — 교회 일정 공유 캘린더

> 용도: AI 디자인 생성 웹 도구(Claude 디자인 / v0 / Lovable 등)에 붙여넣어 고충실도 UI를 생성하기 위한 프롬프트. 특정 도구에 매이지 않게 작성됨.
> 사용법: 한 번에 붙여넣어도 되고, 마지막 줄대로 **디자인 시스템 + 앱 셸 + 캘린더 메인부터** 생성한 뒤 화면을 하나씩 이어가면 결과가 더 안정적이다.
> 기준: 기능/화면 정의서 v1.1. 1차 적용 대상 = 대학부. 영어 지시문 + 한국어 UI 텍스트(한글 문구는 그대로 유지).
> ⚠ 로그인 확정(2026-06-17, 와이어프레임): **이름 + 소속 팀 선택 + 비밀번호**(소셜 로그인 미지원, 이메일 로그인 아님).

---

## 프롬프트 (복사해서 사용)

```
Build a high-fidelity, responsive web UI for a Korean church shared-calendar app
called "교회 일정 캘린더". Keep all UI copy in Korean exactly as written below — do
not translate it. Desktop-first, but fully responsive (desktop 3-pane / tablet
2-pane / mobile single column + drawer). Generate a clickable, multi-screen design
with a reusable component library and consistent styles.

PRODUCT
Multiple admins post events; each member sees only the events of the categories
(teams/groups) they belong to. Roles are assigned PER CATEGORY — the same person can
be a normal member in one category and an admin in another. First rollout target =
대학부 (college ministry).

DESIGN SYSTEM (define as reusable tokens/styles)
- Font: Pretendard (fallback Noto Sans KR). Comfortable line-height 1.5 for Korean.
- Colors: Navy #1F3864 (titles/accents), Brand Blue #2E75B6 (primary buttons/links/
  active), Light Blue #D9E2F3 / #EDF2FB (selected rows, subtle fills),
  Surface #FFFFFF, App bg #F5F7FB, Border #E3E8F0, Text #1F2937, Muted #6B7280.
  Semantic: success #2E7D32, warning #C77700, danger #C62828.
- Category colors (one per category; used on event chips + filter dots):
  #2E75B6, #2E7D32, #C77700, #7E57C2, #0097A7, #D81B60, #5D4037, #455A64.
- 8px spacing grid; radius 8px (12px for modals); soft shadows on cards/modals only;
  inputs & buttons min height 40px.
- Components: Button (primary/secondary/ghost/danger), Input, Select, Date-time
  picker, Checkbox, Toggle, Tag/Chip (category color), Avatar, Modal, Toast, Table,
  SideNav item, Top app bar, Empty state, Inline form error.

APP SHELL (all signed-in screens)
- Top bar: left = logo + "교회 일정 캘린더"; view toggle (월 / 주 / 일 / 목록),
  "오늘" button, ◀ ▶ with period label (e.g. "2026년 6월"); right = profile avatar
  (dropdown: 마이페이지, 로그아웃).
- Left sidebar (drawer on mobile): "내 카테고리" list — each row = colored dot +
  name + checkbox. Below a divider, a "+ 일정" button shown ONLY when the user has
  create permission. Example categories: 대학부, 찬양팀, 전체 공지.
- Auth screens (login/register) use a centered card, no shell.

SCREENS TO GENERATE
1) 로그인 — 이름, 팀 선택(가입 시 선택한 팀 목록 드롭다운), 비밀번호(show/hide),
   "로그인" 버튼, 링크 "비밀번호 찾기"·"회원가입". 소셜 로그인 미지원(이메일 로그인
   아님). Show an inline error state ("이름·팀·비밀번호가 일치하지 않습니다").
2) 회원가입 — 이름, 팀 선택(대학부·대학1부·찬양팀·전체 공지·새가족부), 생년월일(date),
   연락처, 이메일, 비밀번호, 비밀번호 확인. 개인정보 수집·이용 동의(필수, 전문 표시 —
   수집항목·이용목적·보유기간·제3자 미제공·거부권) 체크박스 + "가입하기". Invite
   variant: 소속 카테고리 프리필(읽기 전용), 칩 "초대받은 그룹: 대학부".
3) 캘린더 메인 (★core) — full app shell + Month view 7-col grid (요일 일·월·화·수·
   목·금·토, 오늘 강조). Events render as rounded chips tinted by category color with
   time+title (e.g. "19:00 대학부 금요모임"); "+2 더보기" overflow. Sidebar category
   filter toggles visibility. Also produce a 목록(agenda) view variant. Empty state:
   "표시할 일정이 없습니다".
4) 일정 등록/수정 모달 — 제목(필수), 시작/종료 일시 + "종일" 토글, 반복(없음/매일/
   매주/매월/사용자 지정), 카테고리(멀티선택 — 사용자가 ADMIN인 카테고리만 선택
   가능), 장소(placeholder "예: 예수홀, 센터"), 설명, 첨부파일. Footer: "취소" /
   "저장" (수정 모드엔 "삭제"). 반복 수정·삭제 확인: "이 일정만 / 이후 모든 일정 /
   전체 일정".
5) 일정 상세 — 제목 + 카테고리 색상 칩, 일시·장소·설명·등록자. 권한 있으면 "수정"·
   "삭제", 없으면 읽기 전용.
6) 마이페이지 — 프로필(이름·연락처·이메일(읽기전용)·"비밀번호 변경"), 내 소속
   카테고리(각 항목에 내 역할 태그 일반/관리자), 알림 설정(앱/이메일 토글 — "추후 제공").
7) 관리자·카테고리 관리 — 카테고리 트리(상·하위, 예: 대학부 > 대학1부), 색상·공개범위·
   구성원 수, "+ 카테고리 추가", 생성/수정 폼(이름·색상·상위·공개범위 토글 비공개/공개).
8) 관리자·구성원 관리 — 카테고리 선택 → 구성원 표(이름·이메일·카테고리 내 역할
   드롭다운 일반/관리자·제거), "+ 구성원 추가"(사용자 검색·다중선택).
9) 관리자·사용자 관리 — 검색·상태 필터, 표(이름·이메일·상태 배지·가입일·글로벌 역할·
   작업), "+ 사용자 초대".
10) 관리자·권한 관리 — 역할(일반 사용자/카테고리 관리자/전체 관리자/최고 관리자)별
   권한 매트릭스(체크박스): 일정 등록, 일정 수정/삭제, 카테고리 관리, 구성원 관리,
   사용자 관리, 시스템 설정.

CROSS-CUTTING
- Permission-based visibility: [+ 일정], 수정/삭제, /admin/* appear only for the
  relevant role (per-category for member/admin; system-wide for 전체/최고 관리자).
  Member sees a read-only calendar.
- Korean date format ("2026년 6월 19일 (금)"), 오전/오후 time.
- Provide states: hover, focus, disabled, loading(skeleton), empty, error.
- Responsive: desktop ≥1024 (3-pane) / tablet 768–1023 (collapsible sidebar) /
  mobile <768 (drawer + single column, default to 주/목록 view).
- Reuse the same tokens/components across every screen.

START with the design system + app shell + 캘린더 메인 screen, then generate the
remaining screens one by one.
```
