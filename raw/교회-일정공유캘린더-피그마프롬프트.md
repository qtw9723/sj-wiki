# Figma Make Prompt — Church Shared Calendar (교회 일정 공유 캘린더)

> How to use: Paste the **Master Prompt** into Figma Make first to establish the project, design system, and app shell. Then paste each **Screen Prompt** one at a time to generate individual frames. Keep all UI copy in Korean exactly as written.
> Source of truth: 기능/화면 정의서 v1.1. 1차 적용 대상 = 대학부(College ministry).

---

## 0. Master Prompt (paste this first)

Design a clean, modern, trustworthy web app for a **Korean church shared-calendar service**. The product lets multiple admins post events and lets members see only the events of the categories (teams/groups) they belong to. All UI text must be in **Korean** (do not translate the Korean strings I provide). Design **desktop-first** but make every screen **responsive** (desktop 3-pane, tablet 2-pane, mobile single column with a drawer).

Brand & tone: calm, friendly, organized, faith-community appropriate. Not corporate-cold, not flashy. High readability.

**Design system — define as reusable styles/components:**

- Color tokens:
  - Primary / Navy `#1F3864` (titles, primary text accents)
  - Brand Blue `#2E75B6` (primary buttons, links, active states)
  - Light Blue `#D9E2F3` and `#EDF2FB` (selected rows, subtle fills)
  - Surface `#FFFFFF`, App background `#F5F7FB`, Border `#E3E8F0`
  - Text `#1F2937` (body), Muted `#6B7280`
  - Semantic: success `#2E7D32`, warning `#C77700`, danger `#C62828`
  - **Category colors** (each category gets one; used for event chips & filter dots): `#2E75B6`, `#2E7D32`, `#C77700`, `#7E57C2`, `#0097A7`, `#D81B60`, `#5D4037`, `#455A64`
- Typography: use **Pretendard** (fallback: Noto Sans KR / system sans). Scale: H1 24–28, H2 18–20, body 14–15, caption 12–13. Comfortable line-height (1.5) for Korean.
- Spacing: 8px base grid (8/12/16/24/32). Generous touch targets (min 40px height for inputs/buttons).
- Radius: 8px (cards, inputs, buttons), 12px (modals). Soft shadow for cards/modals only.
- Components to create as reusable: Button (primary / secondary / ghost / danger), Text input, Select, Date-time picker, Checkbox, Toggle, Tag/Chip (category color), Avatar, Modal, Toast, Table, Side navigation item, Top app bar, Empty state, Form field with inline error.

**App shell (for all authenticated screens):**

- Top app bar: left = logo + product name "교회 일정 캘린더"; center/left = calendar view toggle (월 / 주 / 일 / 목록), "오늘" button, ◀ ▶ month navigation; right = profile avatar with dropdown (마이페이지, 로그아웃).
- Left sidebar (collapsible to a drawer on mobile): category filter list (each row = colored dot + category name + checkbox), a divider, and a primary "+ 일정" button shown **only when the user has create permission**.
- Main content area to the right.

**Auth screens (login/register/forgot)** use a centered single-column card on a soft background, no app shell.

**Permission model to reflect in UI:** Roles are assigned **per category** — the same person can be a normal member in one category and an admin in another. Show admin-only controls ([+ 일정], edit/delete, member management) **only for categories where the user is an admin**. System-wide roles (전체 관리자 / 최고 관리자) see admin menus. Front-end hiding is UX only.

After establishing this, wait for individual screen prompts. Generate a reusable component library and consistent styles across all frames.

## 1. Screen — 로그인 (Login) · `/login`

Create a centered auth card (max ~400px wide) on the soft background.
- Title: "로그인", subtitle: "교회 일정 캘린더에 오신 것을 환영합니다".
- Fields: 이메일 (placeholder "이메일 주소"), 비밀번호 (placeholder "비밀번호", with show/hide toggle).
- Primary button (full width): "로그인".
- Divider "또는".
- Social login buttons: "Google로 계속하기", "카카오로 계속하기" (kakao = yellow `#FEE500`).
- Links row: "비밀번호 찾기" (right), "회원가입" (left, "계정이 없으신가요? 회원가입").
- States to show: default, focused field, inline error ("이메일 또는 비밀번호가 올바르지 않습니다").
- Responsive: full-width card with side padding on mobile.

## 2. Screen — 회원가입 / 초대 수락 (Register) · `/register`

Centered auth card, slightly taller.
- Title: "회원가입".
- Fields (in order): 이름, 생년월일 (date picker), 연락처 (phone), 이메일, 비밀번호, 비밀번호 확인.
- Helper note under the form: "이름·생년월일·연락처는 추후 교회 시스템 연동을 위한 식별 정보로 사용됩니다." (small, muted).
- Checkbox: "이용약관 및 개인정보 수집·이용에 동의합니다" (with links).
- Primary button: "가입하기".
- Invite variant note: when arriving via an invite link, 이메일 and 소속 카테고리 are pre-filled and read-only; show a small chip "초대받은 그룹: 대학부".
- States: validation errors (e.g., "비밀번호가 일치하지 않습니다", "이미 사용 중인 이메일입니다").

## 3. Screen — 캘린더 메인 (Calendar) · `/calendar`  ★ core screen

Use the full app shell. This is the primary screen — make it polished.
- Top app bar with view toggle (월 / 주 / 일 / 목록), "오늘", ◀ ▶, current period label (e.g., "2026년 6월"), profile.
- Left sidebar: heading "내 카테고리", list of categories each with a colored dot + name + checkbox (checked = visible). Example items: ▢ 대학부, ▢ 찬양팀, ▢ 전체 공지. Below a divider, a prominent "+ 일정" button (only if permitted).
- Main: **Month view** calendar grid (7 columns Sun–Sat, Korean weekday headers 일·월·화·수·목·금·토). Today's cell highlighted. Events render as **rounded color chips** tinted by their category color, showing time + title (e.g., "19:00 대학부 금요모임"). Multiple events per day; "+2 더보기" overflow indicator.
- Clicking an event chip opens the Event Detail (screen 5). Clicking an empty day (if permitted) opens Event Create (screen 4).
- Also produce the **목록(Agenda) view** variant: events grouped by date in a vertical list, each row = color dot + time + title + category name.
- Empty state: "표시할 일정이 없습니다" with a subtle illustration.
- Responsive: tablet = collapsible sidebar; mobile = sidebar becomes a hamburger drawer, default to 주/목록 view, calendar scrolls.

## 4. Screen — 일정 등록/수정 모달 (Event Create/Edit Modal)

A centered modal (~560px) over a dimmed backdrop.
- Title: "일정 등록" (or "일정 수정").
- Fields:
  - 제목 (text, required)
  - 일시: 시작 날짜/시간, 종료 날짜/시간; "종일" toggle
  - 반복: dropdown (없음 / 매일 / 매주 / 매월 / 사용자 지정) + 종료 조건
  - 카테고리: multi-select chips — **only categories where the user is an admin are selectable**
  - 장소: text (placeholder "예: 예수홀, 센터")
  - 설명: textarea
  - 첨부파일: file upload (optional)
- Footer buttons: secondary "취소", primary "저장"; in edit mode also a danger "삭제".
- Recurring edit/delete confirmation sub-dialog: options "이 일정만 / 이후 모든 일정 / 전체 일정".
- States: validation (required 제목/일시), disabled save until valid.

## 5. Screen — 일정 상세 (Event Detail)

A modal or right-side panel.
- Header: event title + a category color chip (e.g., "대학부").
- Rows: 일시 (with calendar/clock icons), 장소, 설명, 등록자.
- If the user can edit (admin of that category or system admin): show "수정" and "삭제" buttons; otherwise read-only.
- Clean key-value layout with icons.

## 6. Screen — 마이페이지 (My Page) · `/my`

App shell, main content = settings-style sections in cards.
- Section "프로필": 이름, 연락처, 이메일 (read-only), "비밀번호 변경" button.
- Section "내 소속 카테고리": list of categories the user belongs to, each with the user's role in that category as a small tag (일반 / 관리자).
- Section "알림 설정": toggles for 앱 알림 / 이메일 알림 + reminder timing select (mark as "추후 제공" / coming soon, slightly de-emphasized).

## 7. Screen — 관리자: 카테고리 관리 (Admin · Categories) · `/admin/categories`

App shell with an admin context. Page title "카테고리 관리".
- Left: a category **tree** (parent → child, e.g., 대학부 > 대학1부) with color dot, member count.
- Primary action button "+ 카테고리 추가".
- Right or modal: create/edit form — 이름, 색상 (color picker from the category palette), 상위 카테고리 (select), 공개 범위 (toggle: 비공개=소속자만 / 공개=전체).
- Table or list columns: 이름, 색상, 공개/비공개, 구성원 수, 작업(수정/삭제).
- Access note: visible to 전체 관리자 이상.

## 8. Screen — 관리자: 구성원/소속 관리 (Admin · Members) · `/admin/members`

Page title "구성원 관리".
- Category selector at top (choose which category to manage).
- Member table: 이름, 이메일, 카테고리 내 역할 (dropdown: 일반 / 관리자), 작업(제거).
- Button "+ 구성원 추가" → opens a user search/multi-select dialog to assign people to this category.
- Note: a 카테고리 관리자 can only manage members within their own category.

## 9. Screen — 관리자: 사용자 관리 (Admin · Users) · `/admin/users`

Page title "사용자 관리". (전체 관리자 전용)
- Search bar + filters (상태: 전체 / 활성 / 정지).
- User table: 이름, 이메일, 상태(활성/정지 badge), 가입일, 글로벌 역할(태그), 작업(역할 변경 / 정지 / 초대 재발송).
- Button "+ 사용자 초대" → invite-by-email dialog with category pre-assignment.

## 10. Screen — 관리자: 권한/역할 관리 (Admin · Roles) · `/admin/roles`

Page title "권한 관리". (최고 관리자 전용)
- List of roles (일반 사용자 / 카테고리 관리자 / 전체 관리자 / 최고 관리자) with description and user count.
- Permission matrix: rows = permissions (일정 등록, 일정 수정/삭제, 카테고리 관리, 구성원 관리, 사용자 관리, 시스템 설정), columns = roles, cells = checkboxes (editable for 최고 관리자).

## 11. Cross-cutting requirements (apply to all screens)

- **Korean weekday/format:** dates in Korean ("2026년 6월 19일 (금)"), 24h or "오전/오후" time as appropriate.
- **Permission-based visibility:** [+ 일정], 수정/삭제, /admin/* entries appear only for the relevant role; Member sees a read-only calendar.
- **States:** design default, hover, focus, disabled, loading (skeleton), empty, and error for key components.
- **Accessibility:** sufficient color contrast, clear focus rings, adequate touch targets, keyboard-navigable.
- **Responsive breakpoints:** desktop ≥1024 (3-pane), tablet 768–1023 (collapsible sidebar), mobile <768 (drawer + single column, calendar defaults to week/agenda).
- **Consistency:** reuse the component library and color/typography tokens defined in the Master Prompt across every frame.
