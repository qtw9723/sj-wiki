## 크롤링이란?

웹에서 데이터를 자동으로 수집하는 것.

## 방식 4가지

### 1. RSS + feedparser
- RSS: 사이트가 새 글 목록을 XML로 공개하는 규격
- feedparser 라이브러리로 파싱
- 빠름, 서버 부담 없음, 합법적
- 단점: RSS 제공 사이트만 가능 (약 30~40%)

```python
import feedparser, requests
response = requests.get("https://openai.com/blog/rss.xml")
feed = feedparser.parse(response.content)
for entry in feed.entries:
    print(entry.title, entry.link, entry.summary)
```

### 2. API
- 사이트가 공식 API를 제공하면 JSON으로 바로 수신
- 예: 네이버 뉴스 검색 API
- NaverSearch MCP가 이 방식을 래핑해서 제공

```python
response = requests.get(
    "https://openapi.naver.com/v1/search/news.json",
    headers={"X-Naver-Client-Id": "...", "X-Naver-Client-Secret": "..."},
    params={"query": "LLM", "display": 10}
)
data = response.json()
```

### 3. HTML 파싱 (BeautifulSoup)
- RSS/API 없는 사이트에서 HTML을 직접 파싱
- CSS 선택자로 원하는 요소 추출
- 단점: 사이트 구조 바뀌면 코드 수정 필요

```python
from bs4 import BeautifulSoup
soup = BeautifulSoup(response.text, "html.parser")
articles = soup.select("div.article-card h2 a")
for a in articles:
    print(a.text, a["href"])
```

### 4. 브라우저 자동화 (Playwright / Selenium)
- JS 렌더링 사이트는 requests로 빈 HTML만 옴
- 실제 브라우저를 코드로 조종
- 느리고 무거우나 모든 사이트 대응 가능

```python
from playwright.sync_api import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://example.com")
    page.wait_for_selector(".article")
    articles = page.query_selector_all(".article")
```

## 방식 비교

| 방식 | 난이도 | 속도 | 적용 범위 |
|------|--------|------|-----------|
| RSS + feedparser | 쉬움 | 빠름 | RSS 제공 사이트 |
| API | 쉬움 | 빠름 | API 제공 사이트 |
| BeautifulSoup | 중간 | 중간 | 대부분 |
| Playwright | 어려움 | 느림 | 전부 (JS 포함) |
