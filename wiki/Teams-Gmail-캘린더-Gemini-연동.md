> [!tip] 핵심 takeaway
> 🧠 회사(M365/Teams)에서 나오는 건 결국 **메일**과 **일정** 둘뿐 → 각각 구글로 흘려보낸 뒤 **Gemini가 한곳(Gmail+구글캘린더)에서 읽게** 하는 게 통합의 핵심.
> 🧠 확정 설계 = **메일은 A1(Gmail POP3 가져오기) + 일정은 B1(Outlook 캘린더 게시→구글 구독)**, **읽기 전용 단방향**. 답장·일정 수락은 계속 Outlook/Teams에서 한다.
> 🧠 **모든 것의 전제는 회사 테넌트 정책**(POP·외부전달·캘린더 게시 허용 여부). 막히면 대체안으로 한 단계씩 내려가는 식으로 진행한다.

> [!note] 확정 사항 (2026-06-15, 사용자 확인)
> - **허브 계정**: 개인 Gmail `qyw9723@gmail.com` (※ `qtw9723`는 git 저장소 소유자명일 뿐, 메일 주소 아님)
> - **통합 범위**: 읽기/보기 단방향 (양방향 동기화·발송 안 함)
> - **메일 경로**: ✅ **A2 = Outlook 자동 전달(Forwarding)** — A1(POP)은 **앱 비밀번호 미제공(기본 인증 차단)**으로 불가 판정. OWA 전달 토글 켜짐 확인.

## 전체 구조 🧠

```
[M365 메일함]      ──(A1: Gmail POP3 가져오기)──▶ [Gmail] ─────┐
[Outlook 캘린더]   ──(B1: 캘린더 게시 ICS 구독)──▶ [구글 캘린더] ─┴─▶ [Gemini 앱]
[Teams 회의 초대]  ──(B2: 메일 .ics 자동추가)────────────────────┘   @Gmail · @Google캘린더
```

회사에서 나오는 두 흐름(메일·일정)을 구글에 모은 뒤, Gemini가 Gmail·구글 캘린더를 읽어 통합 관리한다.

---

## 구성요소 A — 메일을 Gmail로

### A1. Gmail "다른 계정 메일 가져오기"(POP3) — ❌ 이 환경에선 불가 🧠
> ⚠️ **2026-06-15 판정**: POP 토글 자체는 켜져 있었으나, M365 보안정책상 **앱 비밀번호가 제공되지 않아(기본 인증 차단)** Gmail POP 인증이 불가 → A2로 전환. 아래는 참고용 기록.

Gmail이 회사 메일함을 주기적으로 끌어와 **Gmail 안에 쌓아둔다.** 검색·라벨·Gemini 읽기에 가장 적합.

1. Gmail → ⚙️ **설정 → 모든 설정 보기 → 계정 및 가져오기**
2. **"다른 계정에서 메일 확인" → 메일 계정 추가**
3. 회사 메일 주소(`sjpark@mindwareworks.com`) 입력
4. POP 서버 정보 입력 (M365 기본: 서버 `outlook.office365.com`, 포트 `995`, **SSL 사용**)
5. 가져온 메일에 라벨(예: `회사`)을 붙이도록 설정

- 단점: ⚠️ 회사가 **POP를 꺼뒀으면 불가**. 가져오는 주기가 **최대 ~1시간 지연**. POP는 보통 받은편지함만 가져옴(폴더 구조 X).
- ⚠️ M365는 보안정책상 **기본 인증(POP) 차단 + MFA**가 흔함 → 그 경우 "앱 비밀번호"가 필요하거나 POP 자체가 막혀 A2로 내려가야 함.

### A2. Outlook 자동 전달 — ✅ 채택 (실사용 경로) 🧠
받은 메일을 Gmail로 즉시 forward. 거의 실시간.

- ✅ OWA → **설정 → 메일 → 전달** → "전달 사용" 체크 + `qyw9723@gmail.com` 입력 → **"전달된 메시지 사본 보관"** 체크 → 저장. **2026-06-15 토글 켜짐 확인.**
- 또는 **설정 → 메일 → 규칙**에서 조건부 전달 규칙 생성(특정 메일만 거를 때)
- 단점: 🧠 전달이므로 폴더/일부 메타데이터는 단순화됨. 회사가 추후 외부 전달을 막으면 끊길 수 있음(현재는 허용).

## 구성요소 B — 일정을 구글 캘린더로

### B1. Outlook 캘린더 게시(ICS) → 구글 캘린더 구독 — ✅ 완료 (정상 표시 확인 2026-06-15) 🧠
읽기 전용 목표에 가장 맞는 방식. Outlook이 내 캘린더를 **ICS 링크로 공개**하고, 구글 캘린더가 그 링크를 **URL 구독**한다.

1. OWA → **설정 → 캘린더 → 공유 및 권한(또는 "캘린더 게시")**
2. 내 캘린더를 **게시(Publish)** → 권한은 "모든 세부정보 보기" → **ICS 링크** 복사
3. 구글 캘린더 → 왼쪽 "다른 캘린더" **+ → URL로 추가** → ICS 링크 붙여넣기
4. 회사 일정이 별도 캘린더로 읽기 전용 표시됨

- 단점: 🧠 **구글의 URL 구독 갱신이 느리다(최대 ~24시간 지연).** → 방금 잡힌 회의가 바로 안 뜰 수 있음. 이 지연을 **B2가 보완**한다.
- ⚠️ 테넌트가 "캘린더 게시"를 꺼뒀으면 불가 → 그 경우 B2만으로 운용.

### B2. 메일로 온 초대(.ics)를 구글 캘린더가 자동 등록 — 보완 🧠📄
A(메일)가 동작하면 Teams/회의 초대 메일(.ics 첨부)이 Gmail에 들어오고, 구글 캘린더 설정으로 **버튼 없이 자동 등록**할 수 있다. B1의 24시간 지연을 메우는 즉시성 보완.

- 📄 구글 캘린더 → **설정 → 이벤트 설정 → "내 캘린더에 초대 추가" → 모든 사람**
- 📄 **"대체 이메일 주소로 전달된 초대에 응답 허용"** 체크(전달 경로일 때)
- 📄 Gemini 텍스트 분석: 첨부 없이 본문만 있어도 날짜·시간·링크를 읽어 "Add to Calendar" 제안(정확도 약 92~95%). 단 🧠 일반 Gmail은 제한적, **Google One AI Premium/Workspace**에서 풀 지원.

> 🧠 B1은 "전체 캘린더를 통째로", B2는 "새 초대를 즉시" — 둘을 같이 켜두면 전체성 + 실시간성을 모두 확보.

## 구성요소 C — Gemini로 통합 읽기 🧠
A·B로 Gmail과 구글 캘린더에 데이터가 모이면, **Gemini 앱에서 확장(`@Gmail`, `@Google 캘린더`)을 켜고** 자연어로 통합 조회한다.

- 예: "이번 주 회사 일정 요약해줘", "안 읽은 회사 메일 중 일정 관련만 뽑아줘"
- ⚠️ 개인 Gmail에서 Gemini의 Workspace 확장은 **Google One AI Premium 구독**일 때 가장 안정적. 무료 Gmail은 기능이 제한될 수 있음(확인 필요).
