## 🤖 에이전트 / 코딩 도구

### Codex + GPT-5.5 실사례 — Nextdoor · Notion 📄
*2026-06-09 · OpenAI Blog*

**Nextdoor**: Codex로 재현 어려운 버그를 추적하고, 플랫폼 간(iOS·Android·Web) 개발을 동시 진행.
**Notion**: 스펙을 "원샷"으로 코드화. AI Voice Input 웹 버전 개발. 소규모 팀의 엔지니어링 파워를 곱절로.

> 🧠 "스펙 → 원샷 코드" 패턴은 [[Cogi-POC-Generator]]의 "자연어 → Dialog JSON" 방향과 정확히 같다. POC를 빠르게 키울 때 Codex를 붙이는 방식 참고할 것.

### OpenEnv — 에이전트 강화학습 오픈소스 환경 📄
*2026-06-08 · HuggingFace Blog*

오픈소스 커뮤니티가 에이전트 RL(강화학습) 환경인 OpenEnv를 지지. 에이전트가 환경과 상호작용하며 스스로 학습하는 인프라.

> 🧠 [[에이전트-자동화-도구]]의 `/goal`(완료 조건까지 자동 반복)과 발상이 같은 흐름 — 에이전트가 스스로 반복·개선하는 구조.

### 에이전트가 HF Spaces 2개를 체이닝해 3D 파리 갤러리 구축 📄
*2026-06-09 · HuggingFace Blog*

에이전트가 Hugging Face Spaces 두 개를 도구처럼 순서대로 연결(chaining)해 "3D 파리 갤러리"를 자동 생성한 사례 (agents.md 활용). 도구 두 개의 조합으로 복잡한 결과를 만들어내는 멀티스텝 에이전트 패턴.

> 🧠 [[Cogi-POC-Generator]]의 "자연어 입력 → 시나리오 JSON 생성 → 챗봇 미리보기"처럼 스텝을 체이닝하는 구조와 같은 발상. 더 넓게는 [[내-MCP-커넥터-환경]]에서 MCP 도구를 조합해 쓰는 방식, [[에이전트-자동화-도구]]의 멀티에이전트 오케스트레이션과 같은 계열 — 직접 만든 도구(크롤러·웹앱)도 에이전트의 체이닝 대상이 될 수 있다.

## 🌐 오픈소스 / 새 모델

### Gemma 4 12B — 구글, 통합 멀티모달 오픈소스 모델 📄
*2026-06-09 · Google DeepMind Blog*

> 🧠 [[Cogi-POC-Generator]]에 로컬 LLM을 붙여볼 때 후보 모델. 멀티모달이라 챗봇 시나리오에 이미지 컨텍스트 추가 가능성도.

### Gemini 3.5 Live Translate — 실시간 음성 번역 📄
*2026-06-10 · Google DeepMind Blog*

거의 실시간으로 자연스러운 음성 번역. Google AI Studio, Google Translate, Google Meet에 적용.

### Cohere North Mini Code — 코딩 특화 소형 모델 📄
*2026-06-10 · HuggingFace Blog*

Cohere의 개발자용 첫 번째 코딩 특화 소형 모델.

### 코드 스위칭(이중언어) ASR 벤치마크 📄
*2026-06-10 · HuggingFace Blog*

프론티어 ASR 모델을 코드 스위칭(한국어↔영어 혼용 등) 음성에서 평가한 벤치마크.
> 🧠 회사 챗봇에서 한/영 혼용 입력을 다루는 경우 직접 관련.

### Cohere Oracle Cloud — OpenAI 모델을 엔터프라이즈 클라우드 약정으로 📄
*2026-06-11 · OpenAI Blog*

기업이 Oracle Cloud 약정을 그대로 사용해 OpenAI 모델·Codex에 접근 가능. 엔터프라이즈 AI 도입 장벽 낮추기.

> 🧠 LLM API를 서비스에 붙일 때 클라우드 약정 연계 방식이 확산 중 — 나중에 B2B로 확장할 때 참고 포인트.

### NVIDIA Nemotron 3.5 Content Safety — 커스터마이즈 가능한 멀티모달 안전 모델 📄
*2026-06-05 · HuggingFace Blog*

> 🧠 챗봇 도메인 직결 — 사용자 입력/출력의 유해성 필터링(가드레일)은 기업용 챗봇의 필수 레이어. [[Cogi-POC-Generator]]나 회사 챗봇에 LLM을 붙일 때 "안전 레이어를 별도 모델로 둔다"는 아키텍처 패턴으로 기억해둘 것.
