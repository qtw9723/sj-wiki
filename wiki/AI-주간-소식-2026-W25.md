## 🤖 에이전트·멀티에이전트

### WorkBench 2026: 직장 에이전트 2년 후 재측정 📄
*2026-06-15 · arXiv 2606.13715*

2024년 WorkBench 벤치마크 재측정. 2024년 최고 모델(GPT-4)이 43% 달성, 유해 행동 26%. **2026년 최고 모델: Claude Opus** — 점수 대폭 상승, 단 한계 여전.

> 🧠 에이전트가 2년간 크게 발전했지만 여전히 실수율 존재. 자동화 파이프라인 설계 시 **사람 검토 포인트** 필수.

### MASLab: LLM 멀티에이전트 통합 코드베이스 📄
*2026-06-15 · arXiv 2505.16988*

LLM 기반 멀티에이전트 시스템(MAS) 비교·실험을 위한 **통합 코드베이스** 공개. 다양한 프레임워크를 단일 인터페이스에서 실험 가능.

> 🧠 LangGraph·AutoGen·CrewAI 비교 학습 시 시작점으로 유용. [[에이전트-자동화-도구]]와 연결해 참조할 실습 자료.

### GUI 에이전트의 시각적 메모리 한계 연구 📄
*2026-06-15 · arXiv 2606.14106*

GUI 에이전트가 화면 메모리에 의존할 때 발생하는 실패 패턴 체계적 분석. 경험 기반 메모리 활용 에이전트 도입 시 주의할 실패 케이스 정리.

> 🧠 자동화 스크립트에 GUI 에이전트 도입 전 실패 모드 파악 필수.

### HuggingFace Spaces 에이전트 체이닝 — 3D 파리 갤러리 📄
*2026-06-09 · HuggingFace Blog*

두 개의 HuggingFace Spaces를 에이전트가 체이닝하여 3D 파리 갤러리를 자동 생성한 사례. 에이전트가 외부 도구를 순차 호출하는 패턴 시연.

> 🧠 [[Cogi-POC-Generator]]에서 복수 LLM·도구 체이닝 구현 시 직접 참조 가능한 패턴.

### 게임 이론적 멀티에이전트 제어 — 프롬프트 인젝션 방어 📄
*2026-06-15 · arXiv 2606.10322*

멀티턴 LLM의 컨텍스트 오염(프롬프트 인젝션·포이즈닝) 취약점에 게임 이론 기반 멀티에이전트 제어로 대응하는 연구.

> 🧠 챗봇 시나리오 구성 시 악의적 입력 방어 패턴. 보안 요구사항 있는 CS 챗봇 배포 시 참조.

### Intelligence Entropy — 프로덕션 멀티에이전트 드리프트 원리 📄
*2026-06-17 · arXiv 2606.18065*

LLM 멀티에이전트 시스템이 프로덕션으로 전환 시 비선형적으로 성능이 저하되는 현상을 **Intelligence Entropy(지능 엔트로피)** 원리로 공식화. ADE 안정성 프레임워크 제안.

> 🧠 클라우드 루틴([[웹-크롤링-기초]])처럼 장기 운영 에이전트는 시간이 지날수록 드리프트한다는 경고. 주기적 검증 루틴의 필요성.

### 멀티에이전트 LLM 동시성 이상 검증 탐지·방지 📄
*2026-06-17 · arXiv 2606.17182*

여러 에이전트가 공유 메모리·벡터 인덱스·도구 레지스트리를 동시 접근할 때 발생하는 **동시성 이상(경쟁 조건)**을 검증 기법으로 탐지·방지하는 연구.

> 🧠 멀티에이전트 시스템 설계 시 상태 공유 구조의 안전성 고려 필수.

### Strands Agents + LeRobot — HuggingFace Hub에서 로봇 하드웨어로 📄
*2026-06-17 · HuggingFace Blog*

Amazon Strands Agents와 HuggingFace LeRobot을 연결해 Hub의 모델을 실제 로봇 하드웨어에 바로 배포하는 파이프라인 시연.

> 🧠 에이전트 체이닝이 소프트웨어를 넘어 물리 세계로. [[에이전트-자동화-도구]] 확장 방향.

### ProvenanceGuard — MCP 기반 LLM 에이전트 출처 검증 📄
*2026-06-17 · arXiv 2606.18037*

MCP(Model Context Protocol)로 다양한 소스(검색·API·DB·의료기록)를 호출하는 LLM 에이전트의 **출처 인식 사실성 검증** 프레임워크.

> 🧠 **직접 연관**: [[내-MCP-커넥터-환경]]에서 NaverSearch·KakaoMap·PlayMCP 등 외부 소스를 MCP로 호출 중. 에이전트가 외부 정보를 인용할 때 출처 검증이 핵심.
