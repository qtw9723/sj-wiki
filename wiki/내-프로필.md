> 이 페이지는 위키의 **모든 답변과 takeaway가 기준으로 삼는 관점**이다.
> sangjun은 **경영학과 출신의 실무형 개발자**로, **SM(인프라·유지보수) → NLP 챗봇 도메인**을 거쳤고,
> 지금은 **"비개발 → 개발"로 전환하기 위해 업무 자동화와 LLM 활용**에 집중하고 있다.
> → 새 자료를 정리할 때는 항상 "이게 **업무 자동화 / LLM·챗봇 / 개발자 성장**에 어떻게 연결되는가"를 먼저 본다.

## 한 줄 정의
경영학과 출신 → 웹개발자 → SM 엔지니어 → **NLP 챗봇 회사에서 자동화·LLM으로 개발 역량을 키워가는 실무형 개발자**.

## 커리어 타임라인
1. **시작 — 취미 게임 개발**: 유니티(Unity)로 간단한 게임 제작.
2. **웹개발자 전환**: 자바(Java)를 공부해 웹개발자가 됨.
3. **SM(시스템 매니지먼트) · 약 1년 8개월**
   - 델파이7(Delphi 7) 기반 사내 ERP 유지보수
   - PHP 기반 웹 홈페이지 관리
   - PLC 관련 내부 IT 업무
   - 네트워크 관리
   - → **레거시 유지보수 + 인프라/네트워크**를 폭넓게 다룬 시기.
4. **현재 — NLP 기반 AI 챗봇 회사 (MWW, Mindwareworks)**
   - 자체 개발 NLP 기반 챗봇 솔루션 제공 회사
   - 담당: 솔루션 기반 챗봇 개발(비개발에 가까움), CS
   - **포지션 📄**: 4명뿐인 **CS팀을 돕는 역할**로 배치. CS팀은 단순 응대가 아니라 솔루션 가이드 제작·신버전 테스트·버그 리포팅·서버/서비스 모니터링·신제품 요구사항 정리·해외사업 전반까지 광범위하게 수행 → 인력 부족을 **자동화 웹 도구**로 메움 → 통합 결과물이 [[mailer|CS SmartHub]].
   - 지향: **개발 업무로의 전환** → 주어진 업무를 자동화하며 개발 역량 축적
   - **🟢 전환 실현 중(2026-06)**: 사내 AI 챗봇 프로젝트에 **개발자로 합류**해 기능 분석·개발, API 정리 등 실제 개발 업무를 수행 중. (회사 프로젝트 기밀 상세는 PC 전용 `sj-wiki-work` vault에 분리 보관)

## 강점 / 특성
- **비전공(경영학) 출신**이라 비즈니스·업무 맥락 이해가 빠름 → 자동화할 "진짜 문제"를 잘 찾음.
- **폭넓은 기술 스펙트럼**: 게임(Unity) → 웹(Java/PHP) → 레거시(Delphi) → 인프라(PLC/네트워크) → 현재 NLP/LLM.
- **실무형**: 거창한 이론보다 "내 업무를 실제로 줄여주는가"를 중시.
