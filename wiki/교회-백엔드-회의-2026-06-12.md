## 회의 개요 📄
- **일시**: 2026-06-12 (금)
- **안건**: 백엔드 개발 현황 확인, 개발 방식 협의

## 1. 백엔드 개발 현황 📄
- 메인 스트림(핵심 기능) 개발 **완료**.
- 이후로는 프론트 개발을 진행하면서 필요한 내용이 생기면 **요청 기반으로 추가 대응**.

## 2. 개발 방식 — Git 브랜치 전략 📄
- `main` 브랜치로 머지해 소스를 옮긴 이후에는 **각자 개인 브랜치를 생성해서 작업**하기로 정의.
- 🧠 추론: main = 통합 기준선, 개인 브랜치 → 작업 후 main 머지 흐름(자료에는 머지/리뷰 세부 규칙까지는 명시 없음).

## 3. 백엔드 개발 문서 (app/docs) 📄
- API 명세서(`api-specs`), DB 명세(`db-schema.md`) 등 백엔드 문서는 **`app/docs`** 안에 정리되어 있음.
- 프론트 규약 / 프론트에서 정의하는 부분들도 **md로 정리해 `app/docs`에 함께** 두기로 함. (→ 액션아이템)

## 4. 진행 예정 작업 & 담당 📄
| 작업 | 담당 |
|------|------|
| DB 마이그레이션 | 백은률 |
| 서버 배포를 위한 소스 이전 | 유성진 |

- 앞으로의 개발은 **유성진의 개인 Git에 올라간 소스** 기준으로 진행. 📄
- **이유** 📄: Vercel 배포 시 Git을 public으로 공개해야 하는데, **팀 Git을 public으로 두지 않기 위해** 개인 Git에 올려 서비스.
- **추후** 📄: 서비스 종료 이후 **팀 Git으로 이관** 예정.
- ⚠️ 🧠 유의: 개인 Git이 public이면 소스/문서에 **시크릿(키·토큰)·민감정보가 노출되지 않도록** 환경변수 분리·`.gitignore` 점검 필요(자료엔 명시 없음, 권장사항).
