## 핵심 기능 📄

```
장 체크 (신약 260장) → 10장마다 요소 1개 지급(가중치 랜덤)
                    → 260장 완독 시 특별 요소
요소 배치 → 테마 캔버스에 좌표(%) 저장 (배치취소 없음 — 나무 변동은 회수로만)
팀 캔버스 조회 → 팀별 요소 배치 현황 + 랭킹
```

> 🧠 "나무"는 숲 테마의 요소이며, 바다=바다생물·밤하늘=별로 동일 메커니즘. 아래 **테마 & 요소 시스템** 참고.

## 테마 & 요소 시스템 📄 (2026-06-22 확정)

> [!check] 확정 사항 (2026-06-22, 사용자 확정)
> 다음은 사용자가 구두 확정한 신규 사양이다(📄 = 확정 결정 / 🧠 = 구현 추론·미정).

**1) 테마 3종 동시 운용** 📄

| 테마 | 배치 요소 |
|------|----------|
| 숲 | 나무 |
| 바다 | 바다생물 |
| 밤하늘 | 별 |

- 📄 세 테마를 **동시에 운용**하고, **팀별 최초 로그인자가 선택한 테마**로 진행한다. (2026-06-22 확정 — 팀 단위)
- 📄 DB는 **기존 명칭(`trees`/`species`) 그대로 운용**한다. 단기 프로젝트로 수련회 이후 폐기 예정이라 테마/요소 일반화 마이그레이션은 하지 않는다. (2026-06-22 확정)
  - 🧠 유의: 세 테마를 한 `trees` 테이블에서 `species` 값으로 구분하므로, 바다생물·별도 `species` 문자열로 들어가는 셈. 단기 운용엔 무리 없음. 문제 소지가 보이면 별도 보고.

**2) 요소 지급 — 가중치 랜덤** 📄 (🚧 시연 후 개발)

- 📄 각 요소는 받을 때 **가중치를 둔 확률 랜덤**으로 지급된다(요소마다 등장 확률이 다름).
- 📄 **확률표는 아직 미정 — 추가 논의 필요. 시연(6/26 예상) 이후 개발 예정.** (2026-06-22 확정)

**3) 랭크 & 이펙트** 📄 (🚧 시연 후 개발)

- 📄 각 요소에 **랭크**가 부여되고, **랭크에 맞는 이펙트**를 보여준다.
- 📄 **랭크 단계·이펙트 사양도 미정 — 시연 이후 개발 예정.** (2026-06-22 확정) 확률표와 함께 논의.

## 핵심 설계 결정 📄

> 전체 결정: `app/docs/backend-decisions.md`

| 결정 | 내용 |
|------|------|
| 나무 회수 | **회수함** — 재계산 후 마지막 획득 나무부터 삭제(배치 나무 포함) |
| 체크 API | **권(book) 단위 bulk replace** — 완료 버튼 시 해당 권 전체 교체 |
| 장 등록 UX | **드래그로도 등록 가능** + **여러 권 동시 등록** 지원 (2026-06-22 확정) |
| 좌표 | % 값(0~100), `numeric(5,2)` |
| 배치 | 사용자가 좌표 지정해 배치 |
| 배치취소 | **없음** — 사용자 자발적 배치취소 기능은 만들지 않음. 나무 변동은 **회수(읽기 취소 시)** 로만 발생. (2026-06-22 최종 확정, 직전 "배치취소 허용" 논의는 폐기) |
| z-index | DB 시퀀스로 획득 순서 = z 순서 |
| 챌린지 기간 | 쓰기(체크)만 기간 제약, 조회는 기간 이후에도 허용 |
| 닉네임 | 10자 이내 (프론트 maxLength 제어) |

> [!check] 확정 사항 (2026-06-22 최종, 사용자 확인)
> 📄 **배치취소 기능은 만들지 않는다.** 사용자가 자발적으로 배치를 되돌리는 기능 없음. 나무 변동은 **회수(읽기 취소 시 점수 재계산 → 마지막 획득분부터 삭제)** 로만 일어난다.
> - 📄 직전 "배치취소(재배치) 허용" 논의는 **폐기** — 별도 unplace 엔드포인트/프론트 진입점 불필요.
> - 🧠 따라서 배치 관련 추가 백엔드 작업 없음. `나무 회수` 로직(기존)만 정상 동작하면 됨.
