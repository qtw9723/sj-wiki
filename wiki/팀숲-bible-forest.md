## 핵심 기능 📄

```
장 체크 (신약 260장) → 10장마다 요소 1개 지급(가중치 랜덤)
                    → 260장 완독 시 특별 요소
요소 배치 → 테마 캔버스에 좌표(%) 저장 (배치취소 가능)
팀 캔버스 조회 → 팀별 요소 배치 현황 + 랭킹
```

> 🧠 "나무"는 숲 테마의 요소이며, 바다=바다생물·밤하늘=별로 동일 메커니즘. 아래 **테마 & 요소 시스템** 참고.

## 테마 & 요소 시스템 📄 (2026-06-22 확정)

> [!check] 확정 사항 (2026-06-22, 사용자 확정)
> 다음은 사용자가 구두 확정한 신규 사양이다(📄 = 확정 결정 / 🧠 = 구현 추론·미정).

**1) 테마 3종 동시 운용** 📄

| 테마 | 배치 요소 |
|------|----------|
| 숲 | 나무 |
| 바다 | 바다생물 |
| 밤하늘 | 별 |

- 📄 세 테마를 **동시에 운용**하고, **최초 로그인하는 사람이 선택한 테마**로 진행한다.
- 🧠 ⚠️ 확인 필요: 테마 선택 주체가 **팀별 최초 로그인자**인지 **전체 최초 로그인자**인지 raw에 명시 없음 — 팀 단위 캔버스 구조상 팀별일 가능성. (백엔드/UX 확정 필요)
- 🧠 DB 영향: 테마·요소 종류를 식별하는 컬럼 필요(기존 `trees.species` 일반화 → 요소 타입/테마 구분).

**2) 요소 지급 — 가중치 랜덤** 📄

- 📄 각 요소는 받을 때 **가중치를 둔 확률 랜덤**으로 지급된다(요소마다 등장 확률이 다름).
- 🧠 구현: 가중치 테이블/시드 기반 추첨, 희귀도와 아래 랭크가 연동될 가능성. (확률표 raw에 없음)

**3) 랭크 & 이펙트** 📄

- 📄 각 요소에 **랭크**가 부여되고, **랭크에 맞는 이펙트**를 보여준다.
- 🧠 가중치 랜덤(희귀할수록 낮은 확률) ↔ 높은 랭크 ↔ 화려한 이펙트로 이어지는 게이미피케이션 보상 루프로 추정. 랭크 단계·이펙트 사양은 raw에 없음(디자인·기획 확정 필요).

## 핵심 설계 결정 📄

> 전체 결정: `app/docs/backend-decisions.md`

| 결정 | 내용 |
|------|------|
| 나무 회수 | **회수함** — 재계산 후 마지막 획득 나무부터 삭제(배치 나무 포함) |
| 체크 API | **권(book) 단위 bulk replace** — 완료 버튼 시 해당 권 전체 교체 |
| 장 등록 UX | **드래그로도 등록 가능** + **여러 권 동시 등록** 지원 (2026-06-22 확정) |
| 좌표 | % 값(0~100), `numeric(5,2)` |
| 배치 | 사용자가 좌표 지정해 배치 |
| 배치취소 | **가능** — 배치한 나무를 인벤토리(미배치)로 되돌림. 2026-06-22 디스코드 확정 ⚠️ 번복: 이전엔 "배치 1회성·좌표 수정 불가" |
| z-index | DB 시퀀스로 획득 순서 = z 순서 |
| 챌린지 기간 | 쓰기(체크)만 기간 제약, 조회는 기간 이후에도 허용 |
| 닉네임 | 10자 이내 (프론트 maxLength 제어) |

> [!check] 확정 사항 (2026-06-22, 디스코드, 사용자 확인)
> 🧠 **배치취소(재배치) 허용으로 결정.** 숲에 배치한 나무를 다시 인벤토리로 되돌릴 수 있다. 기존 설계의 "배치 1회성(좌표 수정 불가)" 규칙은 **폐기**.
> - 🧠 백엔드: 배치된 나무를 미배치(`is_planted=false`)로 되돌리는 처리 필요 — 좌표 비우고 인벤토리 복귀. (구현 상세는 raw에 없음 / 백엔드 반영 예정)
> - 🧠 프론트: 배치된 나무에 '배치취소' 진입점 추가 필요.
> - 🧠 `나무 회수`(점수 미달 시 시스템이 삭제)와는 **다른 동작** — 배치취소는 사용자가 자발적으로 인벤토리로 되돌리는 것(나무 자체는 보존).
