## 크롤링 방식 4가지

### 1. RSS + feedparser (현재 사용 중 ✅)
사이트가 새 글 목록을 XML로 공개하는 규격(RSS)을 feedparser 라이브러리로 읽는 방식.

```python
import feedparser, requests

response = requests.get("https://openai.com/blog/rss.xml")
feed = feedparser.parse(response.content)
for entry in feed.entries:
    print(entry.title, entry.link, entry.summary)
```

- **장점**: 빠름, 서버 부담 없음, 합법적, 코드 단순
- **단점**: RSS를 제공하는 사이트만 가능 (전체의 약 30~40%)
- **현재 수집 소스**: Anthropic, HuggingFace, OpenAI, Google DeepMind, Papers with Code, MIT AI

---

### 2. API (현재 사용 중 ✅)
사이트가 공식 데이터 API를 제공하면 JSON으로 바로 수신. [[카카오-Play-MCP]]의 NaverSearch가 이 방식.

```python
response = requests.get(
    "https://openapi.naver.com/v1/search/news.json",
    headers={"X-Naver-Client-Id": "...", "X-Naver-Client-Secret": "..."},
    params={"query": "LLM", "display": 10}
)
data = response.json()
for item in data["items"]:
    print(item["title"], item["link"])
```

- **장점**: 정형화된 JSON, 인증으로 안정적, 가장 깔끔
- **단점**: API를 제공하는 사이트만 가능, 키 발급 필요
- **현재 사용**: NaverSearch MCP가 API 키 관리까지 처리

---

### 3. HTML 파싱 — BeautifulSoup
RSS/API 없는 사이트에서 HTML을 직접 받아 CSS 선택자로 원하는 요소 추출.

```python
from bs4 import BeautifulSoup

response = requests.get("https://example.com/news")
soup = BeautifulSoup(response.text, "html.parser")

articles = soup.select("div.article-card h2 a")  # CSS 선택자
for a in articles:
    print(a.text, a["href"])
```

- **장점**: RSS 없는 사이트도 가능, 설치 간단 (`pip install beautifulsoup4`)
- **단점**: 사이트 구조가 바뀌면 선택자 수정 필요, JS 렌더링 불가

---

### 4. 브라우저 자동화 — Playwright / Selenium
JavaScript로 렌더링되는 사이트는 `requests`로 빈 HTML만 옴. 실제 브라우저를 코드로 조종.

```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://example.com")
    page.wait_for_selector(".article")       # JS 렌더링 완료 대기
    articles = page.query_selector_all(".article")
    for a in articles:
        print(a.inner_text())
    browser.close()
```

- **장점**: JS 사이트, 로그인 필요 사이트도 가능. 모든 사이트 대응
- **단점**: 느림, 무거움, 봇 탐지 우회 필요할 수 있음
- **🧠 연결점**: [[내-MCP-커넥터-환경]]의 `computer-use` / `Claude in Chrome`이 발상이 같음 — 브라우저를 코드로 제어하는 자동화

## 방식 비교

| 방식 | 난이도 | 속도 | 적용 가능 사이트 | 현재 사용 |

| RSS + feedparser | ⭐ 쉬움 | 빠름 | RSS 제공 (~40%) | ✅ |

| Playwright | ⭐⭐⭐ 어려움 | 느림 | 전부 (JS 포함) | 미사용 |

## 추가 학습 포인트

- **robots.txt**: 사이트 루트의 `/robots.txt` — 크롤링 허용 여부 명시. 항상 확인하는 습관.
- **rate limiting**: 요청 간격 조절 (`time.sleep(1)`) — 서버에 부담 주지 않기 위함.
- **User-Agent**: `requests.get(..., headers={"User-Agent": "..."})` — 봇으로 차단 방지.
- **BeautifulSoup 설치**: `pip install beautifulsoup4`
- **Playwright 설치**: `pip install playwright && playwright install`
