> [!tip] 핵심 takeaway
> 🧠 **실제로 올림푸스로 개발 돌릴 때 이 페이지 하나만 보고 시작하면 된다.** 세팅은 대부분 **env 변수 지정 + spec 폴더 넣기**가 전부. 일회성 준비(`.env.local`·`olympus-inbox` 클론)는 **이 PC엔 이미 되어 있음**(2026-07-03 E2E 검증). → 새 프로젝트면 **B(매 실행 세팅)만** 보면 된다.
> 📄 도구 자체 개념·역할은 [[올림푸스-Olympus]], 기획서 쓰는 법은 [[올림푸스-기획요청서-작성요령]].

## 0. 한 줄
📄 `spec/<프로젝트>/`에 기획서 넣고 → **env로 대상·검증·원격 지정** → `bash run.sh`. 나머지(요구분석→개발→verify→승인→머지→배포→아카이브)는 파이프라인이 자동.

## A. 새 환경 부트스트랩 (기기당 1회) 🆕
> **이 맥(주 기기)은 이미 완료 ✅** — 새 PC/노트북에서 처음 세팅할 때만 아래를 따른다.
> 의존성은 최소: **bash · git · python3(표준 라이브러리만, pip 설치 없음) · Claude Code CLI · gh(선택)**. macOS/Linux/Windows(Git Bash·WSL) 동작.

### A-0. 사전 요구 확인
```bash
git --version        # git
python3 --version    # 3.x (notify.py·inbox_imap.py = stdlib만)
claude --version     # Claude Code CLI (없으면 A-2)
gh --version         # GitHub CLI (inbox repo 생성·클론에 편함; 없으면 웹으로 대체)
```

### A-1. 올림푸스 repo 클론
```bash
mkdir -p ~/IdeaProjects && cd ~/IdeaProjects
git clone https://github.com/qtw9723/olympus.git      # private (qtw9723)
```
- 📄 `PROJECTS_DIR` 기본값이 `~/IdeaProjects` — 대상 프로젝트들도 이 아래에 둔다(`spec/<이름>/` → `~/IdeaProjects/<이름>`).

### A-2. Claude Code CLI 설치·인증
- 📄 run.sh가 `claude -p`(헤드리스)로 4역할 세션을 돌린다. 설치 후:
  - GUI 로그인 세션이 있으면 그냥 `claude` 한 번 로그인.
  - **헤드리스/서버(비-GUI)**면 `claude setup-token`으로 토큰 인증(키체인 없이).
- ⚠ 📄 `claude` 플래그(`--session-id`·`--resume`·`--add-dir`·`--model`·`--permission-mode`)는 **CLI 버전마다 다를 수 있음** → 새 기기 첫 실행은 README **B(수동, 터미널 여러 개)**로 한 바퀴 검증 후 자동(`run.sh`) 권장.
- 🧠 macOS 기본 `bash`는 3.2(구버전)라 스크립트가 이에 맞춰 작성돼 있음(빈 배열·`uuidgen` 폴백 처리 완료) → 별도 bash 업그레이드 불필요.

### A-3. gh 인증 (inbox repo 준비용)
```bash
gh auth login        # 또는 이미 인증돼 있으면 생략
```

### A-4. `.env.local` — 알림 + 메일 승인 비밀 (`REMOTE=1` 쓸 때)
```bash
cd ~/IdeaProjects/olympus
cp .env.local.example .env.local
# .env.local 편집: SMTP_APP_PASSWORD 채우기
#   SMTP_SENDER=qyw9723@gmail.com
#   SMTP_APP_PASSWORD=<Gmail 앱비번 16자, 공백 없이>
```
- 📄 이 값 하나가 **알림 발송(SMTP) + 최종 승인 답장 수신(IMAP)** 공용. 값 출처: [[mailer]] Supabase `sender_accounts`(email=qyw9723). `.env.local`은 gitignore(커밋 안 됨).
- 📄 비우면 알림/메일승인만 스킵되고 개발 자체는 됨(최종 승인은 GitHub 경로로 폴백).

### A-5. `olympus-inbox` repo 준비 (블로커·의도질문 GitHub 경로)
```bash
gh repo create qtw9723/olympus-inbox --private        # 이미 있으면 생략
cd ~/IdeaProjects
git clone https://github.com/qtw9723/olympus-inbox.git   # → ~/IdeaProjects/olympus-inbox
```
- ⚠ 📄 **run.sh가 자동 클론하지 않음** — `INBOX_DIR`(기본 `~/IdeaProjects/olympus-inbox`)에 **미리 클론돼 있어야** `REMOTE=1`의 블로커/의도질문 경로가 동작. (최종 승인은 메일이라 inbox 불필요.)

### A-6. 세팅 검증 (스모크)
```bash
cd ~/IdeaProjects/olympus
bash test/run_tests.sh          # → ALL TESTS PASSED (순수 bash·python, 네트워크 X)
# 알림/승인 비밀 확인(발송 안 함, dry-run):
set -a; . ./.env.local; set +a
echo test > /tmp/b && python3 notify.py --subject "smoke" --body-file /tmp/b --dry-run
# IMAP 로그인만 확인(읽기 전용):
python3 - <<'PY'
import os,imaplib
u=os.environ["SMTP_SENDER"]; p=os.environ["SMTP_APP_PASSWORD"].replace(" ","")
m=imaplib.IMAP4_SSL("imap.gmail.com",993); m.login(u,p); print("IMAP OK", m.select("INBOX")[0]); m.logout()
PY
```
- 📄 `ALL TESTS PASSED` + `IMAP OK` 나오면 부트스트랩 완료 → **B(매 실행 세팅)**으로.

> [!note] 자동 위키화 기기와 헷갈리지 말 것
> 🧠 올림푸스 부트스트랩은 [[sj-wiki-launchd-autosync|sj-wiki launchd]]와 무관하다(별개 저장소·별개 목적). 올림푸스는 **수동 실행**(cron/launchd 아님).

## B. 매 실행 세팅 (세션마다) ⭐
### ① 기획서 넣기
- 📄 `~/IdeaProjects/olympus/spec/<프로젝트>/*.md` — **한 프로젝트 = 하위 폴더 하나**(가이드: `spec/_README.md`). 작성 요령 → [[올림푸스-기획요청서-작성요령]].
- 📄 폴더명 `<프로젝트>` = 대상 repo 이름이 된다(`$PROJECTS_DIR/<프로젝트>`, 기본 `~/IdeaProjects/<프로젝트>`).

### ② (재사용 시만) state 초기화
- 📄 이전 프로젝트 state가 남아 있으면 `bash reset.sh` (spec·config·실행로그는 보존, `state/`만 비움).
- 📄 **첫 실행/새 클론은 자동 init** → reset 불필요.

### ③ 실행
```bash
cd ~/IdeaProjects/olympus
TARGET_REPO=~/IdeaProjects/<repo> \        # 여러 spec 폴더 있으면 필수(아래 ⚠)
VERIFY_CMD="<검증 명령>" \                   # 대상 repo의 lint·test·build (심판)
REMOTE=1 POLL_TIMEOUT_SEC=1800 \           # 개입을 폰으로, 답 대기 30분
PERMISSION_FLAG="--dangerously-skip-permissions" \  # 완전 무인(스캐폴딩 포함)
  bash run.sh
```
- 📄 **greenfield/brownfield 자동 판단**: `TARGET_REPO` 경로가 **없으면** greenfield(신규 repo `git init` + 범용 게이트 `templates/verify.sh` 설치), **있으면** brownfield(기존 코드 수정, 작업 브랜치 `olympus/<프로젝트>-<타임스탬프>` 자동 생성, 동작 보존).
- 📄 `VERIFY_CMD` 기본 `bash verify.sh`. 기존 repo면 그 repo 명령으로(`npm run verify`·`pytest -q`·`deno test` 등). greenfield는 안 줘도 범용 게이트 자동 설치.
- ⚠ 📄 **spec 폴더가 여러 개**면 `TARGET_REPO` 미지정 시 **알파벳순으로 오선택**될 수 있음 → 돌릴 프로젝트의 `TARGET_REPO`를 **명시**하거나, 안 쓸 기획서는 `spec/_done/`으로 치워둔다.

## C. 노브 (필요할 때만; 기본값이면 생략) 📄
`config.sh` 기본값으로도 무인 완주. 바꾸고 싶을 때만 env로 덮어쓴다.

| 변수 | 기본 | 용도 |
|---|---|---|
| `REMOTE` | 0 | 1=개입을 폰으로(블로커·의도=GitHub, 최종승인=메일). 끄면 터미널 |
| `POLL_TIMEOUT_SEC` | 21600(6h) | 원격 답 대기 상한. 초과 시 안전 폴백(보류/중단) |
| `PERMISSION_FLAG` | `--permission-mode acceptEdits` | 완전 무인(명령실행·스캐폴딩 포함)은 `--dangerously-skip-permissions` |
| `AUTO_APPROVE` | 0 | 1=최종 승인도 생략(진짜 무인). 승인 게이트 없음 |
| `AUTO_MERGE` | 1 | 승인 시 작업 브랜치를 기준 브랜치로 자동 머지 |
| `DEPLOY_ON_DONE` / `DEPLOY_CMD` | 1 / (빈값) | 머지 후 배포. 규약은 기획서(state/deploy.md)나 `DEPLOY_CMD`. 없으면 끝에 질문 |
| `ARCHIVE_ON_DONE` | 1 | 완료 후 기획서를 `spec/_done/`으로 아카이브 + state 초기화 |
| `TOKEN_WAIT` | 1 | 사용량 한도 걸리면 리셋까지 대기 후 자동 재개(대기 중 토큰≈0). ⚠ 프로세스 죽으면 유실→재실행 |
| `APPROVE_VIA_EMAIL` / `APPROVE_FROM` | 1 / `$SMTP_SENDER` | 최종 승인 메일 답장 경로 / 답장 허용 발신자(위조차단) |
| `GIT_BRANCH` | auto | brownfield 작업 브랜치(auto/none/이름) |
| `MAX_ROUNDS` | 6 | 작업당 Builder↔Critic 최대 라운드 |
| 모델 | metis·critic=opus-4-8, builder=sonnet-4-6 | 비대칭(생성자≠비평가) |

## D. 실행 중 개입 (폰) 📄
잘 쓴 기획서면 개입은 드물다(모호함은 가정으로 진행). 발동하면:
- **블로커 / 의도질문**(자유서술 필요) → 폰 **GitHub 앱**에서 `olympus-inbox/pending/<id>.md`의 `## 답변` 채우고 commit → run.sh가 폴링 감지.
- **최종 머지 승인**(가장 잦음) → 승인 요청 메일에 **답장 첫 줄 `y`**(보류 `n`)만 보내면 감지. (IMAP 불가 시 GitHub 경로로 자동 폴백.) 상세: [[올림푸스-Olympus]] §헤드리스 무인 실행.
- **배포 규약 없음** → 끝에 폰/터미널로 "배포 명령 입력 또는 skip" 질문.
- **Critic 사람 에스컬레이션**(turn=human, 심판변경·보안·스펙 중대위반) → 자동 진행 멈춤 → `state/critic-feedback.md` 확인.

## E. 빠른 시작 체크리스트 (복붙용)
```
□ spec/<프로젝트>/ 에 기획서 넣음 (한 폴더 = 한 프로젝트)
□ (재사용이면) bash reset.sh
□ REMOTE=1 쓸 거면: .env.local 있음 + olympus-inbox 클론 있음  ← 이 PC는 됨
□ 실행:
    cd ~/IdeaProjects/olympus
    TARGET_REPO=~/IdeaProjects/<repo> VERIFY_CMD="<검증>" \
    REMOTE=1 POLL_TIMEOUT_SEC=1800 PERMISSION_FLAG="--dangerously-skip-permissions" \
      bash run.sh
□ 폰: 블로커/의도=GitHub olympus-inbox / 최종승인=메일 답장 y
□ 로그: state/runs/run-*.log
```
