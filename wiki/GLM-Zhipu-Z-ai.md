> [!tip] 핵심 takeaway
> GLM은 중국 **Zhipu AI(Z.ai)**의 오픈소스 LLM 계열로, **"클로드급 코딩 성능을 1/5~1/10 가격에"**가 셀링포인트다. 너에게 의미 있는 두 갈래:
> - **① Claude Code 대체·병행용** → GLM Coding Plan **월 $18(Lite)**부터, Claude Code·Cline 등에 API 물려 쓴다. Anthropic 구독 대비 파격가라 사이드 프로젝트([[올림푸스-Olympus]]·[[개발-위시리스트]])의 토큰 비용 방어책. **올림푸스가 Claude 플랜에서 한도(주당 Opus시간)에 자꾸 걸리는 문제 → §4.5에 Claude 플랜 비교·해법 정리(핵심: GLM API는 주당 캡이 없음).**
> - **② LLM 앱 백엔드용** → [[CogInsight-Generator]]가 지금 OpenAI gpt-4o를 쓰는데, GLM API는 **입력 $0.6 / 출력 $2.2** (플래그십도 $1.4/$4.4)로 훨씬 싸다. **무료 모델(GLM-4.7-Flash 등)**도 있어 실험/POC 부담이 없다.
>
> 🧠 단, 중국 회사 모델이라 **회사(MWW) 데이터·사내 챗봇엔 데이터 거버넌스 확인 전까지 부적합**. 개인/포트폴리오·오픈소스 실험용으로 먼저 보는 게 맞다.

## 1. GLM이 뭔가 (한 줄)
- 📄 **Zhipu AI**(智谱, 칭화대 스핀오프)가 만드는 오픈소스 LLM 패밀리. 해외 브랜드는 **Z.ai**. "GLM = General Language Model".
- 📄 대표 강점: **코딩·에이전트·툴 사용**. GLM-4.6는 Claude Code·Cline·Roo Code·Kilo Code 등 20+ 코딩 툴에 바로 붙여 쓰도록 설계됨.
- 📄 라이선스: 주요 모델(GLM-4.5/4.6 등)이 **MIT 라이선스 오픈소스** → 상업적 사용·셀프호스팅 허용. HuggingFace(`zai-org/GLM-4.6`)·Ollama·OpenRouter로도 배포.

## 4.5 Claude 플랜 vs GLM — 올림푸스 관점 (🧠 핵심)
> [!warning] 왜 [[올림푸스-Olympus]]가 Claude 플랜에서 효율이 안 나오나
> 올림푸스는 **4역할(메티스·헤파이스토스·모모스·헤르메스) Claude Code 세션을 병렬**로 돌린다 → 동시 소비 4배. Claude 구독은 한도가 **"주당 Opus 시간"**으로 잡히는데, Opus는 제일 비싼 모델이라 **가장 빨리 소진**된다. 그래서 "토큰이 너무 적다"고 느끼는 것. (사용자 확인, 2026-07-07)

**Claude 구독 플랜** 📄 (support.claude.com, 2026-07-07)
| 플랜 | 가격/월 | 한도(주당, 대략) | 비고 |
|---|---|---|---|
| Pro | $20 | 가장 낮음 | 올림푸스 4세션엔 부족 |
| **Max 5x** | **$100** | Pro의 5배 | |
| **Max 20x** | **$200** | 최대 **Opus ~40h / Sonnet ~480h**(주당) | 사실상 최상위 |

- 📄 한도는 **5시간 창 + 주당 캡** 이중 구조, 전 모델 공통 캡 + Sonnet 전용 캡 별도. 5시간 캡은 2026-05-06 2배 확대, 주당 한도는 ~2026-07-13까지 +50% 프로모.
- 🧠 병렬 세션은 각 세션이 개별로 카운트 → 올림푸스는 단일 세션 대비 한도를 몇 배 빨리 먹는다. Opus 40h/주는 "1인 다세션 무인 파이프라인"엔 빠듯.

**GLM과의 구조 차이** 🧠 — GLM은 한도를 **Opus시간이 아니라 "5시간당 프롬프트 수"**로 잰다(§4). 병렬·장시간 소비에 훨씬 관대하고 가격도 1/2~1/5.

| | Claude Max 20x | GLM Coding Plan Max | GLM API |
|---|---|---|---|
| 가격/월 | $200 | **$160** | 종량제(입력$0.6/출력$2.2) |
| 한도 방식 | 주당 Opus ~40h | 5h당 ~1,600프롬프트(주 8천) | **캡 없음**(쓴 만큼) |
| 병렬 4세션 | 캡 빨리 소진 | 프롬프트 여유 큼 | 무제한(비용만) |
| 코딩 품질 | 최상(Opus 4.8) | 상(GLM-4.6/5.2, 클로드급 주장) | 동일 |

**올림푸스에 대한 제안 (🧠 판단, 검증 전)**
1. **가장 현실적**: **GLM API 종량제**를 올림푸스 백엔드로 → **주당 캡이 아예 없어** 무인 실행이 한도에 안 걸린다. 4.6 기준 입력 $0.6/출력 $2.2라 하룻밤 대량 실행도 감당 가능. Claude Code에 `ANTHROPIC_BASE_URL`류로 GLM 붙이는 방식(z.ai가 Claude Code 호환 제공).
2. **정액 선호 시**: GLM Coding Plan Max($160) — Claude Max 20x($200)보다 싸고 프롬프트 여유 큼.
3. **품질 이중화**: **심판(verify.sh 통과 판정)·설계(메티스)는 Claude, 반복 구현(헤파이스토스)은 GLM** — 올림푸스가 verify.sh 게이트 구조라 모델 교체가 안전. 어려운 판단만 비싼 모델에 남기고 대량 노동은 싼 모델로.
- ⚠ 회사 코드가 올림푸스에 들어가면 §6대로 중국 모델 데이터 거버넌스 확인 필수. 개인/실험 파이프라인이면 바로 시도 가능.

## 4.6 예상 비용 시나리오 (🧠 가정 기반 — 측정 전)
> [!warning] 이 표의 성격
> **단가는 📄 실측(공식 요금), 실행량은 🧠 가정**이다. 올림푸스 실제 토큰량은 아직 안 재봤으므로, 아래는 "1 표준 실행"을 가정한 추정. **정확히 하려면 Claude Code `/usage`로 실제 run당 토큰을 재서 이 가정에 대입**할 것.

**① 단가표 (per 1M tokens)** 📄
| 모델 | 입력 | 캐시읽기(≈0.1×) | 출력 |
|---|---|---|---|
| Claude Opus 4.8 | $5 | $0.5 | $25 |
| Claude Sonnet 5 | $3 ($2 인트로~8/31) | $0.3 | $15 ($10) |
| Claude Haiku 4.5 | $1 | $0.1 | $5 |
| GLM-5.2(플래그십) | $1.4 | $0.26 | $4.4 |
| **GLM-4.6** | $0.6 | $0.11 | $2.2 |
| GLM-4.5-Air(경량) | $0.2 | $0.03 | $1.1 |
| GLM Flash | 무료 | 무료 | 무료 |

**② "1 표준 실행" 가정** 🧠 — 기획서 1건 → 동작 기능. 4역할×~2h. 캐싱 적용 가정: 신규입력 3M + 캐시읽기 30M + 출력 2M 토큰.

| 백엔드 | 신규입력 | 캐시읽기 | 출력 | **run당 ≈** |
|---|---|---|---|---|
| Opus 4.8 | 3M×$5=$15 | 30M×$0.5=$15 | 2M×$25=$50 | **≈ $80** |
| Sonnet 5(인트로) | $6 | $9 | $20 | **≈ $35** |
| **GLM-4.6** | $1.8 | $3.3 | $4.4 | **≈ $9.5** |
| GLM-4.5-Air | $0.6 | $0.9 | $2.2 | **≈ $3.7** |

→ 🧠 **GLM-4.6는 Opus 대비 run당 약 1/8, Air는 약 1/20.**

**③ 월 사용강도별 총비용** 🧠 (구독=정액, API=위 run당×횟수)
| 월 강도 | Claude Max20x $200 | Claude API(Opus) | GLM Coding Plan | GLM API(4.6) | GLM API(Air) |
|---|---|---|---|---|---|
| 가벼움(10 run) | $200(여유) | ~$800 | **Pro $72**(여유) | ~$95 | ~$37 |
| 중간(20 run) | $200(**캡 근접/초과**) | ~$1,600 | **Max $160**(여유) | ~$190 | ~$74 |
| 무거움(40 run) | **캡 초과→불가** | ~$3,200 | **Max $160**(프롬프트캡 근접) | ~$380 | ~$148 |

**④ 해석 (🧠 판단)**
- **Claude 구독은 "정액이지만 벽이 있다"** — Max 20x도 주당 Opus~40h라 무거운 무인 실행은 중간 강도에서 이미 캡에 부딪힘(= 네가 겪는 문제). 캡을 넘으면 돈을 더 내도 못 쓴다.
- **가장 저렴+무제한 조합**: 무거운 강도라면 **GLM Coding Plan Max $160 정액**이 Claude $200보다 싸고 캡 여유 큼. **캡 자체가 싫으면 GLM API 종량제**(4.6 ~$190, Air ~$74).
- **run당 최저가**: 잡일 역할(헤파이스토스 반복 구현)을 **GLM-4.5-Air나 무료 Flash**로 돌리면 run당 $4 이하. 심판/설계만 Claude로 남기는 이중화(§4.5-3)가 비용·품질 균형 최적.
- ⚠ 표의 절대값은 가정치. **비율(1/8~1/20)은 단가에서 나온 값이라 견고**하니, 실측 후에도 상대적 판단은 유지된다.

## 4.7 예산 제약 해법 — 월 ~$20 이하 (📄 사용자 확정)
> [!note] 확정 사항 (2026-07-07, 사용자 확인)
> **현재 Claude Pro($20) 사용 중이고, 이보다 크게 더 쓸 의향 없음.** → §4.5~4.6의 Max($100~200)·GLM Max($160) 옵션은 **예산 밖으로 배제**. 목표: **월 ~$20 유지하며 올림푸스 캡 문제 해결.**

**핵심 지렛대: GLM 무료 모델(Flash)** 📄 — GLM-4.7-Flash·4.5-Flash는 입력·출력 **전부 $0**. Claude Code 호환으로 붙이면 그 세션은 **추가 비용 0원**.

**$20 이하 옵션 비교** 🧠
| 옵션 | 월비용 | 캡/토큰 문제 | 품질 | 평가 |
|---|---|---|---|---|
| Claude Pro 단독(현재) | $20 | ❌ 4세션 병렬에 부족(=지금 문제) | 최상, 단 벽 | 현상 |
| **Claude Pro + GLM 무료 Flash(잡일 오프로드)** | **$20** | ✅ Claude 부하 급감 | 설계·심판=Opus/구현=중 | **추천 1** |

**추천 (🧠 판단)**
1. **1순위 — Claude Pro 유지 + 무거운 역할을 GLM 무료 Flash로 이관.** 추가 지출 **0원**. 토큰 잡아먹는 **헤파이스토스(반복 구현)**를 GLM Flash에 넘기면 Claude Pro 한도는 설계(메티스)·심판에만 쓰여 벽에 잘 안 부딪힘. verify.sh 게이트가 있어 구현 모델이 싸도 품질 방어됨.
2. **2순위 — Claude 해지하고 GLM Coding Plan Lite($18)로 갈아타기.** 올-GLM, $2 더 쌈. 단 프롬프트 쿼터(~400/주)가 4세션에 충분한지 **테스트 필요**(부족하면 1순위로 복귀).
3. **완전 무료 실험 — 전 역할 GLM Flash.** $0, 캡 없음. 품질은 낮지만 "일단 무인으로 계속 돌아가는지"부터 검증할 때 유용.

⚠ 어느 쪽이든 회사 코드가 올림푸스에 들어가면 §6 데이터 거버넌스 확인 먼저. 개인 프로젝트면 바로 시도 가능.

- **컨텍스트 200K** (GLM-4.5의 128K → 확장), **357B 파라미터 MoE**, **MIT 라이선스**, 출시 2025-09-30.
- 📄 8개 공개 벤치마크(에이전트·추론·코딩)에서 GLM-4.5 대비 전반 상승, DeepSeek-V3.1-Terminus·Claude Sonnet 4와 경쟁 우위 주장.
- 📄 GLM-4.5 대비 **작업당 토큰 ~15% 절감**하면서 복잡 멀티턴 성공률 개선 → 실사용 비용이 이중으로 저렴.
- ⚠ 멀티모달 표기 충돌: llm-stats는 "텍스트+이미지+비디오 입력"이라 하나, 공식 z.ai는 GLM-4.6를 **텍스트 전용**으로, 비전은 별도 **GLM-4.6V**로 분리해 둠. 이미지 처리는 4.6V 계열로 보는 게 안전.

## 6. 너에게의 적용 (🧠 판단)
- **사이드/오픈소스 프로젝트**: [[올림푸스-Olympus]]는 여러 Claude 세션이 병렬로 도는 구조라 토큰 소모가 크다. GLM Coding Plan(월 $18~)이나 GLM API를 백엔드 옵션으로 두면 실험 비용을 크게 낮출 수 있다. (단 올림푸스는 verify.sh 심판 구조라 모델 교체가 비교적 자유로움)
- **[[CogInsight-Generator]] 백엔드**: 현재 OpenAI gpt-4o. GLM API(또는 무료 Flash)로 A/B 해볼 가치. 생성 품질만 받쳐주면 운영비가 급감. ⚠ 단 CogInsight는 사내 POC라 **회사 데이터 거버넌스**부터 확인.
- **회사(MWW) 사내 챗봇**: ⚠ 중국 회사 모델 → 사내 기밀·고객 데이터엔 신중. 셀프호스팅(MIT라 가능)으로 데이터를 외부로 안 내보내는 옵션은 있으나 GPU 비용·검토 필요. 이 판단·상세는 [[내-프로필]] 회사 맥락상 개인 vault엔 원칙만 남긴다.
- **Claude Code 병행**: 지금 주력은 Claude(Opus 4.8, [[Claude-Code-업데이트-동향]]). GLM은 "고빈도 저위험 코딩은 싼 모델, 어려운 건 Claude"식 이중 전략의 저가 축으로 유용.
