## 🔥 빅 이슈

### Instruction Bleed — 프롬프트 모듈 간 교차 간섭 📄
*2026-06-26 · arXiv 2606.26356*

프롬프트로 조립한(prompt-composed) 에이전트 시스템에서 실무자들이 반복 겪는 실패 모드를 공식화: **공유 변수도, 실행 의존성도 없는데 프롬프트 한 모듈을 수정하면 다른 모듈의 행동이 조용히 바뀐다.** 이를 "조합적 행동 누수(compositional behavioral leakage)"로 정의.

> 🧠 **[[Cogi-POC-Generator]] 직격탄**: Cogi는 **LLM 6단계 생성 파이프라인** + DB-구동 규칙엔진이라 프롬프트 모듈이 여러 단계에 걸쳐 있다. "한 단계 프롬프트를 손봤더니 다른 단계 출력이 어긋나는" 현상이 바로 Instruction Bleed. → 파이프라인 단계별로 **회귀 테스트(고정 입력→기대 출력)**를 둬서 한 모듈 수정이 다른 단계를 깨는지 잡아야 한다. W25 [[AI-주간-소식-2026-W25|평가 인식·LegalHalluLens]]의 "오류 유형을 분리 측정하라"와 같은 처방.

### OpenAI Daybreak — 전 조직 보안을 위한 AI 도구군 📄
*2026-06-22 · OpenAI Blog*

OpenAI가 **Daybreak** 이니셔티브로 보안 도구 공개: **Codex Security**(코드 취약점 탐지)와 **GPT-5.5-Cyber**(사이버보안 특화 모델)로 조직이 취약점을 **대규모로 탐지·검증·패치**하도록 지원. 함께 발표한 **Patch the Planet**은 오픈소스 메인테이너가 AI+전문가 검토로 취약점을 찾고 검증·수정하도록 돕는 프로그램.

> 🧠 **방어적 보안 자동화의 흐름**: "AI가 취약점을 찾고→검증하고→패치 제안"은 내가 쓰는 [[Claude-Code-업데이트-동향|Claude Code의 security plugin]]과 정확히 같은 패턴(찾기-검증-수정). [[mailer|CS SmartHub]]·[[Cogi-POC-Generator]] 같은 내 코드베이스에 적용 가능한 워크플로우. "탐지→**검증**→패치"에서 검증 단계를 끼우는 건 W25의 "제안-검증-종합 구조"와 동일.

### GPT-5.6 Sol 프리뷰 — 코딩·과학·사이버보안 강화 📄
*2026-06-26 · OpenAI Blog*

OpenAI가 차세대 모델 **GPT-5.6 Sol** 프리뷰 공개. **코딩·과학·사이버보안** 역량 강화 + 가장 진보한 안전 스택 동반.

> 🧠 모델 세대 교체 신호. [[Cogi-POC-Generator]]는 현재 OpenAI **gpt-4o** 기반 → 코딩·구조화 출력이 강해지면 시나리오 생성 품질 업그레이드 후보. 단, W25 키로(Kiro)·로컬모델 교훈대로 **"최신·최대 모델 ≠ 최적"** — 비용·지연 대비 검증 후 도입.

### 삼성전자, ChatGPT Enterprise + Codex 전사 배포 📄
*2026-06-22 · OpenAI Blog · (6/27 다이제스트만 수록)*

삼성전자가 전 세계 임직원에 **ChatGPT Enterprise와 Codex 배포** — OpenAI의 최대 규모 엔터프라이즈 AI 롤아웃 중 하나.

> 🧠 **사내 AI 도입 레퍼런스**: W25의 네이버 Claude Code·LG CNS 1천명·더존비즈온에 이어, 국내 대기업의 전사 AI 코딩도구 도입 사례 추가. [[내-프로필]] 기준 "사내 챗봇/도구 도입 제안" 시 "삼성도 Codex 전사 배포"라는 근거로 활용.

## 🤖 에이전트·멀티에이전트

### Gemini 3.5 Flash에 컴퓨터 사용 기능 도입 📄
*2026-06-25 · Google DeepMind Blog*

Google이 **저비용·고속 모델인 Gemini 3.5 Flash**에 컴퓨터 사용(computer use) 기능 도입. 화면을 보고 클릭·입력하는 에이전트를 경량 모델로.

> 🧠 **[[웹-크롤링-기초]] 직결**: 지금은 Playwright 스크립트로 정해진 셀렉터를 조작하지만, computer-use 에이전트는 **화면을 보고 스스로 조작**한다. [[mailer|CS SmartHub]]의 Playwright 챗봇 모니터링을 "셀렉터 깨지면 멈추는" 방식 대신 시각 기반으로 보강할 후보. Flash급 저비용이라 반복 모니터링에 현실적.

### Semantic Early-Stopping — 반복 에이전트 루프의 의미 기반 종료 📄
*2026-06-26 · arXiv 2606.27009*

Writer(초안)·Critic(수정) 같은 멀티에이전트 루프는 보통 **고정 반복 횟수(max_iterations)**로 끝난다. 이건 "답이 충분히 좋아졌는지"를 못 보는 **구문적 kill-switch**라고 지적, 답의 의미적 수렴을 보고 멈추는 방법 제안.

> 🧠 [[에이전트-자동화-도구]]의 workflows에서 "loop-until-dry/until-good" 패턴과 정확히 같은 문제의식. Cogi 6단계 생성에서 "재생성 몇 번"을 고정값이 아니라 **품질 수렴 기준**으로 거는 근거.

### CUGA — 경량 하네스 위 실전 에이전트 앱 24종 📄
*2026-06-23 · HuggingFace Blog (IBM Research)*

**CUGA**: 가벼운 하네스 위에서 동작하는 **실전 에이전트 앱 두 다스(24개) 예제** 공개.

> 🧠 무거운 프레임워크 없이 에이전트 앱을 만드는 레퍼런스. [[Cogi-POC-Generator]]·[[에이전트-자동화-도구]] 학습/실습 자료.

### LLM이 모호한 지시를 이해하도록 — 2-모델 분업 📄
*2026-06-26 · MIT News*

로봇이 집·공장에서 일하도록, MIT가 **한 LLM은 사용자의 모호한 지시를 명확히 하고(clarify), 다른 LLM은 관련 없는 정보를 무시(filter)**하는 2-모델 분업 접근 제시.

> 🧠 **챗봇 설계 패턴**: "사용자 발화 정제 → 핵심만 추출"은 CS 챗봇([[Cogi-POC-Generator]]·[[mailer|CS SmartHub]])에서 모호한 고객 문의를 처리하는 구조와 동일. 단일 프롬프트에 다 욱여넣지 말고 **역할을 분리**하라는 근거 — 위 Instruction Bleed의 모듈 분리와도 통함.

### Murakkab — 멀티스텝 AI 워크플로우의 속도·에너지 최적화 📄
*2026-06-25 · MIT News*

**Murakkab**: AI 애플리케이션을 구동하는 멀티스텝 워크플로우의 설계·배포를 최적화해 속도·에너지 효율 개선.

> 🧠 에이전트 파이프라인이 길어질수록 비용·지연이 쌓인다 → 워크플로우 자체를 최적화하는 시스템 연구. [[에이전트-자동화-도구]]의 다단계 workflows 비용 관리 관점.

### Experience Compression Spectrum — 에이전트의 메모리·스킬·규칙 통합 📄
*2026-06-26 · arXiv 2604.15877 (v2)*

장기·다중 세션 에이전트에서 누적 경험 관리가 병목이 됨. 에이전트 **메모리 시스템**과 **스킬 발견**을 하나의 스펙트럼(메모리↔스킬↔규칙)으로 통합해 재사용 가능한 지식을 추출.

> 🧠 내가 쓰는 [[Claude-Code-업데이트-동향|Claude Code]]의 **메모리·스킬** 구조와 직접 닿는 연구. 위키 운영처럼 "한 번 정리해 축적하고 재사용"하는 [[내-프로필|이 vault의 철학]]과도 같은 방향(경험을 매번 재추출하지 않고 규칙·스킬로 압축).

## 의외의 연결점

> 🧠 **이번 주 관통 주제 = "프롬프트/에이전트를 모듈로 짜되, 모듈 간 간섭을 관리하라"**: Instruction Bleed(모듈 수정이 옆 모듈을 깬다) ↔ MIT 2-모델 분업(역할 분리) ↔ Semantic Early-Stopping(루프 종료 기준) — 모두 **"멀티 모듈/멀티 에이전트를 안전하게 조립하는 법"**. [[Cogi-POC-Generator]] 6단계 파이프라인 설계에 직접 적용.
> 🧠 **"AI 보안 자동화"가 한 카테고리로 부상**: Daybreak(Codex Security·GPT-5.5-Cyber) + GPT-5.6 Sol(사이버보안 강화) + Patch the Planet — 내가 쓰는 [[Claude-Code-업데이트-동향|Claude Code security plugin]]과 같은 "탐지→검증→패치" 워크플로우가 업계 표준이 되는 중.
> 🧠 **"비용 최적화 모델 선택"이 계속 강화**: 로컬모델 PR 트리아지(무료) + Gemini 3.5 **Flash** 컴퓨터 사용 + Murakkab(워크플로우 효율) — W25 키로의 "작고 저렴한 모델" 메시지가 W26에 더 굵어짐. [[Cogi-POC-Generator]]·[[웹-크롤링-기초]] 비용 설계 근거.
