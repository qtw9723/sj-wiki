> [!tip] 핵심 takeaway
> **한 번 깊게 익히면 7개 프로젝트에 전부 통하는 너의 "주력 무기 세트"** — React 19 + Vite 8 + Tailwind 4 + Supabase.
> [[내-프로필]]의 "개발자 성장" 목표에서 **여기에 학습을 집중하는 게 ROI가 가장 높다.** 스택이 통일돼 있어 한 프로젝트의 개선이 곧 전 프로젝트의 개선이 된다.
> 약점은 **단조로움** — 모든 앱이 같은 스택이라, 도구 업그레이드/패턴 변경 시 일괄 작업이 잦다. 이건 [[에이전트-자동화-도구]]로 자동화하기 좋은 조건.

## 표준 스택
**프론트엔드**
- React 19 (전 프로젝트)
- Vite 8 (빌드)
- Tailwind CSS 4 (스타일)
- lucide-react (아이콘), react-router-dom (라우팅)

**백엔드/데이터**
- Supabase (PostgreSQL + Auth + Edge Functions) → [[parking]]에서 공유 관리. Edge Functions(Deno)는 [[Cogi-POC-Generator]](생성·학습·인증 20개 함수)·[[mailer]]·[[notepad]] 등이 직접 사용.
- Express.js ([[mailer]], [[schedule-reporter-kakao]] — Node 서버 풀스택)
- Nodemailer (메일 발송)
- 🧠 [[Cogi-POC-Generator]]는 더 이상 프론트 전용이 아니라 **Supabase Edge Functions 백엔드를 갖춘 풀스택**(LLM 호출=OpenAI gpt-4o를 엣지함수 안에서).

**자주 쓰는 라이브러리**
- @dnd-kit (드래그앤드롭) — notepad·mailer·schedule-reporter 공통
- zod (스키마 검증) — 특히 [[Cogi-POC-Generator]]의 Dialog JSON 검증
- react-hook-form, Radix UI / shadcn (Cogi)
- react-markdown + remark-gfm ([[notepad]])

## 학습/개선 포인트
- **zod**는 챗봇 시나리오 검증([[Cogi-POC-Generator]])과 LLM 출력 검증에 핵심 → 우선 깊게. 실무 API/DB 스키마를 검증 대상으로 삼으면 효과적.
- 스택 통일의 단점(반복 업그레이드)은 [[Claude-Code-업데이트-동향]]의 dynamic workflows로 "전 프로젝트 일괄 마이그레이션" 자동화 가능.

> 개인 프로젝트 스택 외의 도구·MCP 커넥터(Cowork·카카오 Play 등)는 [[내-MCP-커넥터-환경]] 참고.
