## 7. ⭐ 순수 Lua 코어의 MSW 밖 실행 (P0, 최우선) — 자동검증 전략의 핵심
> 이게 되면 올림푸스가 게임 규칙을 사람 확인 없이 검증할 수 있다. 안 되면 검증 전략을 바꿔야 한다.
- [ ] MSW **의존 API를 전혀 안 쓰는** 순수 Lua 모듈 작성(예: 간단한 전투 계산 함수 `damage(a,b) return ... end`).
- [ ] 그 모듈을 MSW 안에서 `require`(또는 MSW 방식)로 **불러와 실행**되는지.
- [ ] 같은 `.lua` 파일을 **MSW 밖 표준 Lua 인터프리터**(로컬 `lua`/`luajit`)로 실행해 보기 → 그대로 돌아가나?
  - 안 되면: MSW가 쓰는 Lua 버전/변형(5.1? 커스텀?) 확인. `require`·파일 분리 방식 확인.
- [ ] MSW에서 스크립트를 **여러 파일/모듈로 분리**해 서로 불러오는 방법 확인(코어를 여러 모듈로 쪼갤 수 있어야 함).
- 확인 목적: "규칙=순수 Lua 코어, MSW=바인딩" 분리 + **MSW 밖 CI 시뮬 테스트** 가능성 판정.
- 판단 기준(셋 중 하나로 결론):
  - 🟢 **A**: 순수 Lua가 MSW 밖 표준 인터프리터로 그대로 실행 → 코어 CI 시뮬 그대로 채택.
  - 🟡 **B**: 표준 밖 실행은 안 되지만 MSW 안에서 모듈 분리·자체 테스트 러너 실행 가능 → 헤드리스 대신 MSW 내 테스트로.
  - 🔴 **C**: 둘 다 어려움 → 검증 전략 재설계(로직을 더 단순화하거나 사람 검증 비중↑).
