# 05. 개발 태스크 (MVP 분해)

> 순서 = 의존성 순. **T1~T9(순수 Lua 코어)는 MSW 없이 올림푸스가 완주·검증 가능**. T10~는 MSW 손검증(04.5) 이후 착수. 각 태스크는 완료 시 `verify.sh`(코어 테스트)가 green이어야 한다.

## A. 순수 Lua 코어 (MSW 독립 — 올림푸스 주 개발 대상)

### T1. 프로젝트 골격 + 테스트 하네스
- 코어 폴더 구조, 경량 Lua 테스트 러너(busted 등), `verify.sh`가 테스트 실행.
- **AC**: 더미 테스트 1개 green, `verify.sh` 종료코드 0.

### T2. 시드 RNG + 유틸
- 결정론 RNG(시드→시퀀스), 셔플, 안전 정수 연산.
- **AC**: 같은 시드→같은 시퀀스/셔플 재현. 단위테스트 통과.

### T3. 데이터 로더 + 스키마 검증
- 카드/몬스터/직업/인카운터/맵/밸런스 데이터(03) 로드 + 필수필드·참조 무결성 검증(존재하지 않는 monsterId/cardId 참조 시 에러).
- **AC**: 샘플 데이터셋 로드 성공, 깨진 참조는 명확한 에러.

### T4. 효과 DSL 해석기 (02.6)
- 키워드 원자 전부 구현(deal_damage/gain_block/summon/heal/apply_status/draw/gain_energy/add_card/exhaust_self/if).
- **AC**: 각 키워드 단위테스트(대상 해석·수치 적용) 통과.

### T5. 전투 상태 + 더미 관리 (02.1, 02.5)
- 상태 구조, 드로우/버림/셔플/소멸, 손패 상한, **카드 집합 보존 불변식**.
- **AC**: 드로우/리셔플 반복 후 카드 id별 개수 보존. 불변식 assert 통과.

### T6. 턴 해소 엔진 (02.3, 02.4)
- 턴 시작(블록0·에너지·드로우), 카드 플레이(비용·효과), 턴종료 해소(아군 공격→적 의도→정리→의도 재결정), 피해/방어.
- **AC**: 시나리오 테스트(정해진 손패·시드로 예상 최종 상태 일치). 결정론 리플레이 해시 일치.

### T7. 적 AI 의도 (02.3, 03.2)
- 결정론 의도 패턴(순환/시드), 공개·실행.
- **AC**: 고정 시드로 의도열 재현, 실행 결과 일치.

### T8. 직업 레이어 (02.8)
- 전사/마법사 패시브 + 시작덱 + 카드풀 색. 밸런스표면 억제(별도 자원 없음) 준수.
- **AC**: 각 직업 런 시작 덱 구성·패시브 발동 테스트 통과.

### T9. 런 진행 + 맵 + 보상 + 메타 (02.9, 02.10)
- 시드 맵 생성, 노드 해소(전투/엘리트/상점/휴식/이벤트/보스), 보상 3중1, 골드, 몬스터 드롭, 도감/해금 상태(영속화는 인터페이스만 — 저장 구현은 바인딩), 런 리셋 규칙.
- **AC**: 시드로 1막 맵 생성·완주 시뮬(직업선택→보스) 크래시 없이 종료. 도감/해금 갱신 검증.

### T10. 자동검증 스위트 완성 (04.4)
- 불변식 시뮬(다중 시드×정책), 결정론 리플레이, 밸런스 승률 리포트.
- **AC**: 전 항목 green, 승률 리포트 출력. 무한루프(≤max_turns) 보장.

## B. MSW 바인딩 (04.5 손검증 이후 — 사람+올림푸스 반자동)

### T11. MSW 개발환경·라이프사이클 확인 반영
- 04.5의 1·2·6 확정 → 어댑터 인터페이스 확정.
- **AC**: MSW에서 코어 1턴이 구동되는 최소 스파이크.

### T12. 전투 화면 UI (02.11)
- 적/의도, 필드 슬롯, 손패·에너지·더미·턴종료. 입력→코어 액션.
- **AC**: 손으로 1전투 플레이 가능.

### T13. 맵/보상/상점/휴식/이벤트 화면
- **AC**: 런 전체를 UI로 플레이 완주.

### T14. 영속 저장 연동 (04.5-4)
- 도감/해금/통계 저장·불러오기.
- **AC**: 재접속 후 도감/해금 유지.

### T15. 에셋 연결 + 제출 빌드 (04.5-5,7)
- 메이플 스프라이트/이펙트 연결, 콘테스트 제출 형태로 게시.
- **AC**: 제출 가능한 MSW 월드 완성.

## 완료 정의 (MVP Done)
- A(T1~T10) 전부 green + B(T11~T15) 완료 → `01.8 성공기준` 충족.
- 사람 확인 지점 최소화: 규칙·밸런스는 자동검증이 지키고, 사람은 방향·UX 감수만.
