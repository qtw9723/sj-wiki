# 03. 콘텐츠 데이터 스펙

> 모든 콘텐츠는 데이터로 정의하고 규칙 엔진이 해석한다. 저장 포맷은 **Lua 테이블 모듈**(권장) 또는 JSON(→ 04에서 MSW 로딩 방식 확인 후 확정). 아래는 논리 스키마 + 예시.

## 3.1 카드 (Card)
```
Card {
  id: string            # 고유 (예: "mon_snail", "warr_taunt")
  name: string
  type: "monster" | "skill" | "spell"
  cost: int             # 에너지
  rarity: "common"|"uncommon"|"rare"|"boss"
  jobTag: "warrior"|"mage"|null   # null=공용
  monsterId: string|null          # type=monster일 때 소환할 몬스터
  effects: Effect[]               # 키워드 DSL (02.6). monster형은 보통 [summon(monsterId)]
  exhaust: bool
  desc: string          # UI 표기용(엔진 판정엔 미사용)
}
```
예시:
```
{ id="warr_taunt", name="도발", type="skill", cost=1, rarity="common", jobTag="warrior",
  effects=[ apply_status(self_front_ally,"도발",1,1), gain_block(self,5) ], exhaust=false }
{ id="mage_fireball", name="파이어볼", type="spell", cost=2, rarity="common", jobTag="mage",
  effects=[ deal_damage(chosen_enemy,10) ], exhaust=false }
{ id="mon_snail", name="달팽이 소환", type="monster", cost=1, rarity="common", jobTag=null,
  monsterId="snail", effects=[ summon("snail") ], exhaust=false }
```

## 3.2 몬스터 (Monster) — 소환수/적 공용 정의
```
Monster {
  id: string
  name: string
  atk: int
  hp: int
  keywords: string[]     # {도발,가시,재생,광폭,비행,...} 닫힌 집합
  aiIntents: Intent[]|null  # 적으로 등장할 때의 의도 패턴(소환수는 단순 공격 기본)
  cardId: string|null    # 이 몬스터를 소환하는 카드(수집 연결)
  tier: "normal"|"elite"|"boss"
  spriteRef: string      # MSW 에셋 참조(확정은 04/에셋 확인)
}
Intent { kind: "attack"|"block"|"buff"|"debuff", value: int, target: string, pattern?: 순환 }
```
- 적 의도는 **결정론 패턴**(고정 순환 또는 시드 선택). 예: 슬라임 = [공격6, 공격6, 방어5] 순환.

## 3.3 직업 (Job)
```
Job {
  id: "warrior"|"mage"
  name: string
  passive: Effect[] | 특수키워드   # 예: warrior = 턴시작 gain_block(self,2)
  startingDeck: cardId[]           # 시그니처 5~7 + 시작 몬스터 카드 몇 종
  cardPool: cardId[]               # 보상으로 등장 가능한 이 직업 카드 풀
}
```

## 3.4 적 조우 / 인카운터 (Encounter)
```
Encounter {
  id: string
  tier: "normal"|"elite"|"boss"
  enemies: monsterId[]   # 진영 구성
  actBudget: int         # 난이도 가중(맵 배치용)
}
```

## 3.5 맵 / 노드 (Map)
```
ActConfig {
  actId: 1
  depth: int             # 층 수 (MVP ~10~12)
  branch: [min,max]      # 층당 갈래
  nodeWeights: { 전투:.., 엘리트:.., 이벤트:.., 상점:.., 휴식:.. }  # 시드 배치 확률
  bossEncounterId: string
}
```

## 3.6 유물/이벤트 (MVP 최소)
- 유물(Relic)은 MVP 범위 밖(차기). 필요 시 `Relic{ id, trigger, effects[] }`로 동일 DSL 재사용.
- 이벤트(Event)는 텍스트 + 선택지 배열, 각 선택지 결과는 Effect[] 또는 보상/패널티(결정론).

## 3.7 MVP 콘텐츠 물량 (초기 목표)
| 항목 | MVP 수량 | 비고 |
|---|---|---|
| 직업 | 2 (전사·마법사) | 각 시그니처 5~7 |
| 몬스터(소환+적) | 20~30 | 노멀 다수 + 엘리트 3~4 + 보스 1 |
| 공용 스펠 카드 | 8~12 | 범용 유틸 |
| 인카운터 | 8~12 | 노멀/엘리트/보스 |
| 이벤트 | 3~5 | 텍스트 선택지 |

## 3.8 밸런스 노브 (한곳에 모음 — ⚙️ 초기값)
```
BALANCE {
  player_maxHp=50, energy_per_turn=3, hand_size=5, hand_cap=10,
  field_slots=3, max_turns=50, rest_heal_pct=0.30,
  reward_card_choices=3, monster_drop_rate=0.5,
  warrior_block=2, mage_spell_bonus=0.5
}
```
> 밸런스 조정 = 이 테이블·데이터 수치 변경. 규칙 코드는 불변. 자동검증(04)이 승률 회귀를 감시.
