# 사용법 — 다른 기기에서 올림푸스로 개발하기

이 폴더(`sj-wiki/specs/MonsterBookSaga/`)가 **기획서 정본**이다. sj-wiki가 git으로 동기화되므로 어느 기기에서든 최신본을 받는다.

## 다른 기기에서 진행 순서
1. `git -C sj-wiki pull --rebase origin main` — 최신 기획서 받기.
2. 이 폴더의 `*.md`(00~05)를 그 기기의 **올림푸스 spec 폴더**로 복사:
   `cp sj-wiki/specs/MonsterBookSaga/*.md ~/IdeaProjects/olympus/spec/MonsterBookSaga/`
3. 올림푸스 `spec/`에 다른 프로젝트 폴더(예: `Mindboard/`)가 있으면 치우고 `bash reset.sh`.
4. `bash run.sh` — 메티스(기획자)가 이 문서를 분석·분해해 개발 시작.

## 주의
- 올림푸스 `spec/`은 gitignore(로컬 입력 전용)라 GitHub에 안 올라간다 → **수정은 항상 이 sj-wiki 정본에서** 하고 push, 다른 기기에선 pull 후 복사.
- MSW 실제 개발방식(Lua 런타임·UI·저장·에셋) 확정 전이므로 `04_기술아키텍처_MSW.md` §4.5 확인 항목을 먼저 손검증한 뒤 T11~ 착수.
- 개발 대상 repo가 MSW 에디터 프로젝트라 올림푸스 범용 verify.sh는 **Lua 코어 테스트 러너로 교체** 필요(04 참조).
