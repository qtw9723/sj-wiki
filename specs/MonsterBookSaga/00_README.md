## 핵심 설계 철학 (전 문서 관통)
- **결정론 우선**: 모든 무작위는 시드 RNG. 같은 입력 → 같은 결과 → 헤드리스 시뮬로 자동검증 가능.
- **데이터 구동**: 카드/몬스터/직업/적은 전부 데이터. 규칙 엔진 1개가 해석. 콘텐츠 추가 = 코드 변경 아님.
- **코어/표현 분리**: 게임 로직은 MSW 의존 없는 **순수 Lua 코어**. MSW는 입력·렌더·저장 바인딩만. → 코어를 CI에서 단독 시뮬 가능.
- **밸런스 표면 최소화**: 직업은 "몬스터를 쓰는 색"만 줄 뿐 별도 자원 시스템을 만들지 않는다.

## ⚠ 실행 전 확인
- 올림푸스 `run.sh`는 `spec/` 하위 첫 폴더(이름 오름차순)를 프로젝트로 잡는다. 현재 `Mindboard/`가 함께 있으면 그게 먼저 잡힌다 → **이 프로젝트를 돌릴 땐 `Mindboard/`를 치우고 `bash reset.sh`**.
- MSW는 일반 git+npm/py repo가 아니라 자체 에디터 프로젝트다. **올림푸스의 범용 verify.sh 게이트를 MSW/Lua용으로 바꿔야** 한다(→ `04_기술아키텍처_MSW.md`). 이 부분은 실제 MSW 개발 방식을 손으로 확인한 뒤 확정한다.
