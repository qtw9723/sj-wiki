# 01. 기획 개요 (PRD)

## 1.1 제품 개요
**메이플 몬스터북 사가**는 메이플스토리 월드(MSW) 위에서 동작하는 **1인용 턴제 덱빌딩 로그라이크**다. 플레이어는 한 직업의 모험가가 되어, 잡은 몬스터를 **소환 카드**로 모으고 직업 스킬 카드와 엮어 덱을 짠 뒤, 분기 맵을 올라가며 층마다 강해지는 적과 보스에 맞선다. 실패하면 처음부터 다시 시작하되(로그라이크), 수집한 몬스터 도감과 해금은 영구히 남아 다음 런이 더 풍부해진다.

## 1.2 배경 / IP 재해석
원작 메이플스토리에는 이미 **몬스터를 사냥하고 "몬스터북(몬스터 카드)"으로 수집**하는 메타가 있다. 본작은 이를 **"수집한 몬스터를 카드로 소환해 내 편으로 싸운다"**로 재해석·확장한다. 사냥의 대상이던 몬스터가 전략 자원이 되는 역전이 핵심 훅이며, 여기에 메이플 **직업군(전사·마법사·궁수·도적·해적)**의 정체성을 얹는다.

## 1.3 목표 (Goals)
1. 시드 기반 **완전 결정론** 턴제 전투가 규칙대로 정확히 동작한다.
2. 플레이어가 **몬스터 소환 카드 + 직업 스킬 카드**로 덱을 구성해 한 판(런)을 끝까지 플레이할 수 있다.
3. 런 도중 몬스터를 처치·수집해 **도감**을 채우고, 다음 런에 반영되는 **영구 해금**이 동작한다.
4. 카드/몬스터/직업/적이 **데이터로 정의**되어 코드 변경 없이 콘텐츠를 늘릴 수 있다.
5. 이 문서만 읽고 올림푸스가 MVP를 완주할 수 있을 만큼 명세가 구체적이다.

## 1.4 심사기준 매핑 (콘테스트 40/40/20)
| 기준 | 대응 설계 |
|---|---|
| **IP 해석·확장 40%** | 몬스터북→소환덱 재해석 + 메이플 5직업군 아키타입 + 원작 몬스터/스킬 자산 활용 |
| **완성도 40%** | 결정론 전투 + 데이터 구동 + 헤드리스 자동검증으로 "버그 없이 끝까지 도는" 안정성 확보 |
| **지속 가능성 20%** | 수집 도감·직업/카드 해금·다회차 조합 다양성 → 반복 플레이 동기 |

## 1.5 핵심 사용자 (Target)
| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| 로그라이크 팬 | 슬더스·몬스터트레인류 경험자 | 덱 조합의 깊이·리플레이성 |
| 메이플 팬 | 원작 IP 애착 | 익숙한 몬스터·직업을 새 방식으로 즐기기 |
| 라이트 유저 | 캐주얼 | 한 판을 짧게, 규칙이 명확하게 |

## 1.6 핵심 사용자 시나리오
1. 직업을 골라 새 런 시작 → 기본 덱(직업 시그니처 카드 + 시작 몬스터 몇 종)을 받는다.
2. 분기 맵에서 다음 노드를 선택 → 전투에 진입한다.
3. 매 턴 에너지로 몬스터를 소환하고 스킬 카드를 써서 적을 처치한다.
4. 전투 보상으로 카드 1/3 선택·골드·몬스터 카드 드롭을 받는다. 상점/휴식/이벤트로 덱을 다듬는다.
5. 막 보스를 처치하면 런 클리어. 사망 시 런 종료하되 도감·해금은 유지되어 다음 런에 반영된다.

## 1.7 범위 (Scope)

### In Scope — MVP (v0.1, "끝까지 되는 게임")
- 결정론 턴제 전투 엔진(에너지·손패·드로우/버림 더미·필드 소환 슬롯·적 의도 표시)
- 카드 3종류: **몬스터 소환 카드 / 직업 스킬 카드 / 공용 스펠 카드**
- **직업 2종**(전사·마법사) — 각 패시브 1 + 시그니처 스킬 카드 5~7종
- **몬스터 20~30종**, 적/엘리트/보스 포함
- **1개 막**: 분기 맵 ~10~12노드 + 보스 1
- 전투 보상(카드 1/3 선택), 골드, 상점·휴식(회복/카드 제거)·간단 이벤트 노드
- **몬스터북 도감** + 영구 해금(카드/직업) 최소 루프
- 프로필 저장/불러오기(영속화)
- 헤드리스 전투 시뮬 + 자동검증 스위트

### Out of Scope (차기)
- 직업 5종 전체·2~3막·유물(relic) 대량·이벤트 다양화
- 멀티플레이·PvP·랭킹(단, MSW는 멀티 지향 → 아키텍처는 열어두되 MVP는 1인용)
- 정교한 애니메이션/연출, 사운드 풀세트
- 밸런스 정밀 튜닝(정식 밸런싱은 콘텐츠 확장 후)

## 1.8 성공 기준 (Success Criteria)
- `05_개발태스크.md`의 완료 체크리스트를 100% 통과한다.
- 동일 시드로 전투를 재생하면 **완전히 동일한 결과**가 나온다(결정론 검증).
- 자동검증 스위트가 불변식을 통과한다: HP 음수 없음, 에너지 보존, 덱/손패 합법성(카드 유실·복제 없음), 전투가 유한 턴 내 종료(무한루프 없음).
- 스크립트된 기본 정책(AI)으로 다회 시뮬 시 **1막 승률이 합리적 범위**(예: 40~70%)에 든다(밸런스 회귀 지표).
- 한 런(직업 선택→1막 보스)까지 플레이가 크래시 없이 완주된다.

## 1.9 가정 및 제약
- **1인용** 기준 설계(MSW 멀티 기능은 MVP에서 사용하지 않음).
- 콘텐츠 양은 "끝까지 되는" 최소치. 방대함보다 **완결성** 우선.
- 실제 MSW 구현 방식(Lua 런타임·UI·저장 API)은 손검증 전이라 `04`의 "확인 필요" 항목에 따라 세부가 바뀔 수 있다. **게임 규칙(02)·데이터(03)는 플랫폼 독립**이라 그대로 유효하다.
