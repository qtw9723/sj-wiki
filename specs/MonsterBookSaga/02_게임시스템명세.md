# 02. 게임 시스템 명세 (결정론 규칙)

> 이 문서의 규칙은 **플랫폼 독립**이다. 모든 수치는 조정 가능한 초기값(⚙️)이며 `03_콘텐츠_데이터스펙.md`의 밸런스 노브로 뺀다. **모든 무작위는 시드 RNG**에서만 나온다.

## 2.1 전투 상태 (Combat State)
한 전투는 다음 상태로 완전히 기술된다:
- **플레이어**: `hp`, `maxHp`(⚙️50), `block`(턴마다 0으로 소멸), `energy`, `energyPerTurn`(⚙️3), `handSize`(⚙️5)
- **더미**: `drawPile`(순서 있음), `hand`, `discardPile`, `exhaustPile`(소멸 카드)
- **필드**: `slots`[⚙️3] — 각 슬롯에 소환된 몬스터 인스턴스 또는 빈칸
- **적 진영**: `enemies`[1..N] — 각 `hp`,`maxHp`,`block`,`intent`(다음 행동, 미리 공개)
- **RNG**: `seed`, `rngState` (전투 내 모든 난수의 유일 원천)

## 2.2 카드 (Card) — 3종류
1. **몬스터 소환 카드**: 비용(energy)을 내고 **빈 필드 슬롯에 몬스터를 소환**. 슬롯이 없으면 낼 수 없다. 소환된 몬스터는 `atk`,`hp`,`keywords[]`를 가진 인스턴스가 된다.
2. **직업 스킬 카드**: 비용을 내고 **즉시 효과** 실행(피해/방어/버프/디버프/드로우/에너지 등). 직업 정체성이 여기서 나온다. 사용 후 버림 더미로(또는 `exhaust`면 소멸).
3. **공용 스펠 카드**: 직업 무관 즉시 효과(범용 피해/방어/유틸).

카드 공통 필드: `id`, `name`, `type`, `cost`, `rarity`, `jobTag`(null=공용), `effects[]`, `exhaust`(bool). `effects`는 **키워드 DSL**(2.6)로만 기술 — 카드마다 임의 코드 금지(결정론·검증성).

## 2.3 턴 흐름 (엄격한 순서)
### 전투 시작
1. 시드로 RNG 초기화. `drawPile` = 덱 전체를 셔플(시드). `discard/exhaust` 비움. 필드 비움.
2. 적 진영 배치 + 각 적의 **첫 의도(intent) 결정**(공개).

### 플레이어 턴 시작
1. `block` = 0.
2. 에너지 = `energyPerTurn`.
3. `handSize`가 될 때까지 드로우(2.5 드로우 규칙).
4. 턴 시작 트리거 효과 발동(예: 소환수 "재생").

### 플레이어 행동 (입력)
- 손패의 카드를 임의 순서로 낸다. 각 카드는 `cost` ≤ 현재 에너지일 때만 낼 수 있다. 낼 때 에너지 차감 → `effects` 순서대로 해석.
- 소환 카드는 대상 슬롯(빈칸) 지정. 스킬/스펠 중 대상 필요한 것은 대상 지정(적/아군 슬롯).
- "턴 종료"를 누르면 행동 페이즈 종료.

### 턴 종료 해소 (결정론 순서)
1. **아군 몬스터 공격**: 슬롯 낮은 번호부터, 각 몬스터가 `atk`만큼 대상(기본 = 가장 앞 적, 또는 키워드 규칙)에게 피해.
2. **적 행동**: 적 번호 순서대로 공개됐던 `intent` 실행(플레이어 또는 아군 슬롯 공격/방어/버프).
3. 상태 정리: 지속효과 감소, `hp<=0` 개체 제거, 죽은 소환수 슬롯 비움.
4. 각 적의 **다음 의도 재결정**(공개).
5. 승패 판정(2.7). 안 끝났으면 다음 플레이어 턴.

## 2.4 피해·방어 규칙
- 피해는 먼저 대상 `block`을 깎고, 남은 만큼 `hp` 차감. `block`은 음수 안 됨.
- 방어(Block) 부여 = 대상 `block` += 값. 플레이어 `block`은 턴 시작 시 0으로 소멸(누적 안 됨, ⚙️).
- 버프/디버프는 **스택 수치**로 표현(예: `근력+2`=아군 공격 +2, `취약`=받는 피해 +50%). 지속턴 or 영구는 데이터로.

## 2.5 드로우/더미 규칙 (카드 유실·복제 없음 — 검증 대상)
- 드로우 시 `drawPile`이 비면 `discardPile`을 시드 셔플해 `drawPile`로 옮긴다. `exhaustPile`은 셔플 대상 아님.
- 손패가 상한(⚙️10)을 넘으면 추가 드로우는 버려짐(over-draw는 discard로).
- **불변식**: (draw+hand+discard+exhaust) 카드 집합 = 덱 정의와 항상 일치(id별 개수 보존).

## 2.6 효과 키워드 DSL (닫힌 집합 — 확장은 데이터+엔진 동시)
카드/몬스터/적 효과는 아래 원자 키워드의 배열로만 기술한다. (엔진이 해석; 임의 Lua 금지)
- `deal_damage(target, amount)` / `gain_block(target, amount)`
- `summon(monsterId, slot?)` / `heal(target, amount)`
- `apply_status(target, status, stacks, duration)`  # status ∈ {근력,취약,약화,재생,도발,가시,…}
- `draw(n)` / `gain_energy(n)` / `add_card(cardId, pile)` / `exhaust_self`
- `target` ∈ {self, front_enemy, all_enemies, chosen_enemy, ally_slot, all_allies}
- 조건부: `if(cond, [effects])` — cond는 닫힌 술어(예: `enemy_count>=2`)
> 새 키워드가 필요하면 **엔진 해석기 + 데이터**를 함께 추가한다(카드별 코드 분기 금지).

## 2.7 승패
- 모든 적 `hp<=0` → 전투 승리 → 보상(2.9).
- 플레이어 `hp<=0` → 런 종료(사망). 도감/해금은 유지.
- **무한루프 방지**: 한 전투 최대 턴(⚙️50) 초과 시 강제 판정(플레이어 패배 처리) + 로그 경고. (정상 밸런스에선 도달 안 함 — 검증 지표)

## 2.8 직업 (Job) 레이어
- 런 시작 시 직업 1개 선택. 직업 = **패시브 1 + 시그니처 스킬 카드 세트**(기본 덱에 포함) + **카드 보상 풀 색**(그 직업 태그 카드가 보상에 등장).
- MVP 2종:
  - **전사(Warrior)**: 패시브 = 턴 시작 시 `block +2`. 색 = 방어/도발/탱커 몬스터 시너지(앞선 아군이 적 공격 유도, 방어 반사 "가시").
  - **마법사(Mage)**: 패시브 = 매 3번째로 내는 스펠의 피해 +50%(⚙️). 색 = 광역 스펠·상태이상(취약/약화)·마법형 소환수 강화.
- **밸런스 표면 억제 규칙**: 직업은 별도 자원/게이지를 만들지 않는다. 오직 (패시브 1) + (카드 풀 색)으로만 차별화한다.

## 2.9 런 구조 (로그라이크)
- **맵**: 시드 생성 분기 그래프. 노드 타입 = `전투`·`엘리트`·`이벤트`·`상점`·`휴식`·`보스`. 위→아래(또는 아래→위) 한 방향, 매 층 2~4갈래.
- MVP: **1막**, ~10~12노드, 막 끝 **보스 1**.
- 노드 해소:
  - 전투/엘리트 승리 → **카드 보상 3장 중 1 선택**(또는 건너뛰기) + 골드 + 확률적 **몬스터 카드 드롭**(처치한 몬스터의 카드).
  - 상점 → 골드로 카드 구매/카드 제거.
  - 휴식 → HP 회복(⚙️최대 30%) 또는 카드 1장 제거 중 택1.
  - 이벤트 → 텍스트 선택지(결정론 결과, 데이터 정의).

## 2.10 메타 진행 / 수집 (지속성)
- **몬스터북 도감**: 전투에서 마주치거나 처치한 몬스터는 도감에 "발견"으로 기록. 카드를 획득하면 "수집"으로 승격.
- **영구 해금**: 런을 거치며 특정 조건(예: 특정 보스 처치, 도감 N종 달성)으로 **새 카드/직업**이 해금되어 이후 런의 보상 풀·시작 선택지에 추가.
- **프로필 영속화**: 도감·해금·통계(플레이 수·클리어 수)는 세션 간 저장(→ 04 저장 API).
- 로그라이크 원칙: 런 내 덱/골드/HP는 런 종료 시 리셋. 도감/해금만 영구.

## 2.11 화면 / UX 흐름 (개략 — 시각 세부는 MSW에서 확정)
1. **타이틀** → 새 런 / 도감 / 통계.
2. **직업 선택** → 시작 덱 미리보기.
3. **맵** → 노드 선택.
4. **전투 화면**: 상단 적(의도 아이콘), 중앙 필드 슬롯(소환수), 하단 손패·에너지·덱/버림 더미·턴종료 버튼.
5. **보상** → 카드 3중 1 선택.
6. **상점/휴식/이벤트** 팝업.
7. **결과**(승리/사망) → 도감·해금 갱신 표시 → 타이틀.
> UI 레이아웃·연출은 MSW UI 시스템 확인 후 확정(→ 04). 규칙 상 필요한 정보 표시 요건만 위에서 고정.
